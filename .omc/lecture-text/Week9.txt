The git commands
31

”What files have I changed since my last commit?”

===== PAGE 32 =====

===== PAGE 33 =====
revert vs. reset — the short version
33
•
`git revert` — creates a new commit that undoes the old one. History preserved. Safe.
•
”Undo my last commit but keep the history.”
•
`git reset` — moves the pointer backward. History lost. Dangerous on shared repos.
•
Rule: Just do git revert.

37

===== PAGE 38 =====

38
•
You may see `worktree` in the Claude Code status bar — with a tooltip: "Work in an isolated copy of the repo"
•
A git worktree is a second working directory linked to the same repo
•
Your repo, but in a separate folder — so changes there cannot break your current work

Claude Code creates a worktree automatically when it runs a task that might touch many files

You do not create or delete worktrees yourself — Claude Code manages them

===== PAGE 58 =====
The concierge MVP — validate before you build
58
•
우동균(Team10) did something important: no app, just text messages
•
Manually checked if users completed their workouts, one by one
•
Found: aggressive reminders work short-term, cause churn long-term
•
This is a concierge MVP — do the thing manually for one real user first
•
The pattern:
•
Pick one core user action your product is supposed to enable
•
Do it manually for one real person — no code, no UI
•
Watch what happens. Note where they get confused or fail
•
Cost: zero engineering time. Output: real behavioral data
•
Build the feature only after you know the manual version works

===== PAGE 59 =====
How to run a Level 2 observation
59
•
Find one person who matches your target user (not a teammate, not a close friend)
•
Give them your product or prototype. Say: "Try to do X. I'll just watch."
•
