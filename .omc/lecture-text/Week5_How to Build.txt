===== PAGE 6 =====
Terminal Basics
Mac — Terminal
$ cd ~/Documents/my-project
Navigate to folder
$ ls
List files
$ mkdir new-folder
Create folder
$ npm install
Install dependencies
$ npm run dev
Start app locally
$ claude
Launch Claude Code
Windows — CMD
C:\Users\[id]> cd ~\Documents\my-project
Navigate to folder
C:\Users\[id]\my-project> dir
List files
C:\Users\[id]\my-project> mkdir new-folder
Create folder
C:\Users\[id]\my-project> npm install
Install dependencies
C:\Users\[id]\my-project> npm run dev
Start app locally
C:\Users\[id]\my-project> claude
Launch Claude Code
Most commands are identical — the key differences are: ls vs dir, / vs \, and the prompt symbol.

===== PAGE 10 =====
When Things Go Wrong
You don't need to understand the error. Just follow this loop.
1
COPY
Select the entire
error message
in your terminal
2
PASTE
Paste it into
Claude Code
or Codex
3
FOLLOW
AI explains the fix.
Apply it. Repeat
if needed.
Repeat until it works. Each cycle gets you closer — AI learns from the context.
This is exactly how professional developers debug too — they just Google faster.
