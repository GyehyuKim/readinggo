## 1. 문제

현재 참새는 **문장 저장 직후** 질문 카드가 등장한다.
읽기 흐름(몰입 상태) 중간에 성찰을 강제 → 둘 다 얕아짐.

---

## 2. 원칙

> **읽기(몰입)와 성찰(대화)은 다른 정신 상태다. 섞지 않는다.**

```
읽는 중      →  저장만  (마찰 제로, 흐름 유지)
읽기 종료 후  →  참새 등장  (성찰 집중, 감정이 살아있을 때)
```

---

## 3. 변경 전/후

| | 현재 (v7.3) | 변경 후 (v7.4) |
|---|---|---|
| **참새 트리거** | 문장 저장 직후 | 읽기 세션 종료 시 |
| **저장 시 피드백** | 참새 질문 카드 | 토스트만 ("문장 저장됨 💬", 0.8초) |
| **선택지** | 없음 | 저장 / 지금 대화 (부수 옵션) |
| **참새 질문** | 저장한 문장마다 | 세션에서 저장한 문장 중 **하나만** |

---

## 4. 읽기 종료 트리거 정의

> ⚠️ **미구현 (드리프트 정정 2026-07-09)**: 아래 트리거는 코드에 없다 — ReadingMode 폐기(#505)로 호스트가 사라졌다.

읽기 세션이 끝나는 경우 3가지 모두에서 참새 등장(설계 의도):

| 트리거 | 조건 |
|---|---|
| **✕ (나가기)** | 목표 페이지 미달성으로 나가기 |
| **체크인 완료** | 목표 페이지 달성 → 체크인 |
| **완독** | `isComplete=true` → Ceremony 이전 |

> 완독 시: 참새 → 별점·소감(Ceremony) 순서. 참새 스킵 가능.

---

## 5. 세션 내 문장이 여러 개일 때 선택 알고리즘

1. 코멘트(감상)가 달린 문장 우선
2. 없으면 가장 마지막 저장 문장
3. 세션에서 저장한 문장이 0개면 참새 미등장

## 6. UX 상세

### 6.1 저장 시 — 토스트만

```
[문장 저장됨 💬]     ← 0.8초 후 자동 소멸
```

읽는 중에는 저장 버튼만 노출(v7.5, #438) — AI 대화로 가는 '💬 지금 대화' 부수 옵션은 **제거**.
읽기(몰입)와 성찰(대화)을 끝까지 분리한다.

### 6.2 읽기 종료 후 — 참새 카드 (말풍선 채팅, #435)

```
┌─────────────────────────────┐
│  오늘 28p 읽었어요 ✨         │
│  (134 → 162p)               │
│                             │
│  오늘 문장 2개 저장          │
│  ─────────────────────────  │
│  " 완전한 행복이란           │
│    존재하지 않지만... "       │
│                             │
│  🐦 ┌──────────────────┐    │
│     │ 왜 이 문장이었어요? │    │  ← 참새 질문(좌측 말풍선)
│     └──────────────────┘    │
│              ┌─────────────┐│
│              │ 생각을 써보세요…││  ← 내 답(우측 말풍선, 입력 후)
│              └─────────────┘│
│                             │
│  [ 답하고 이어가기 ] [건너뛰기]│
└─────────────────────────────┘
```

- 읽기 요약(오늘 몇 p) + 참새 질문이 한 화면에
- 질문/답 모두 **말풍선**(좌=참새, 우=나) — 채팅 대화 느낌(#435)
- "건너뛰기" 항상 보임 — 페널티 없음
- 답하면 후속 질문 1개 가능 (멀티턴 §327 연동)

### 6.3 둥지 귀환

```
┌─────────────────────────────┐
│  🔥 13일째                  │
│  +45 XP                    │
│                             │
│  오늘의 한 문장이            │
│  서재에 쌓였어요 (총 23개)   │
│                             │
│  [ 둥지로 ]                 │
└─────────────────────────────┘
```

XP는 읽기(페이지) + 문장 저장 + 참새 답변 모두 획득.
참새 건너뛰어도 읽기·저장 XP는 유지.

## 7. companion.md 변경 내용

companion.md §2 코어 루프를 아래로 교체:

**현재:**
```
한 문장 저장 → 참새 "왜 이 문장 남겼어요?" → 답(또는 스킵) → 대화 축적
```

**변경 후:**
```
읽는 중: 한 문장 저장 (토스트만, 흐름 유지)
    ↓
읽기 종료
    ↓
참새: 세션 문장 중 하나 선택 → "왜 이 문장이었어요?"
    ↓
답변 (또는 건너뛰기)
    ↓
(선택) 후속 질문 1개 — 멀티턴
    ↓
대화 축적 → 내 한 문장 카드에 표시
```

## 8. 구현 변경 범위

> ⚠️ **미구현 (드리프트 정정 2026-07-09)**: 아래 변경은 **적용되지 않았다** — `nest.js`에 `startCompanion()`은 존재하지 않으며, ReadingMode 폐기(#505)로 읽기 종료 핸들러 자체가 없다.

| 파일 | 변경 |
|---|---|
| `nest.js` | 문장 저장 직후 `startCompanion()` 호출 제거 → 읽기 종료 핸들러로 이동 |
| `nest.js` | 읽기 종료(✕ / 체크인 / 완독) 시 세션 저장 문장 수집 + `startCompanion()` 호출 |
| `components.js` | 저장 UI: 참새 질문 카드 → 토스트 + "지금 대화" 부수 버튼 |
| `nest.js` | 읽기 종료 화면: 오늘 요약 + 참새 카드 통합 뷰 |

## 9. 비목표 (Non-goals)

- 참새 없는 읽기 모드 토글 (설정 복잡도 ↑, Phase 2 이후)
