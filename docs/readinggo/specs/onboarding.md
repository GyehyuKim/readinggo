## 4. 유저 시나리오 — 신규 가입 여정

각 단계는 화면 단위이며, **Phase 0 데모에서 모두 클릭으로 확인 가능해야 한다.**

### A. 진입 (비로그인)

- **빈 상태 = 약속+기능 예고 카드 (#1134, 2026-07-03)**: 참새 헤일로 + 헤드라인 "하루 한 쪽, 한 문장이면 돼요" + 기능 예고 3줄(rgIcon 모노라인 — ① 한 문장→둥지 성장 ② 재키 AI 대화 ③ 같은 책 타인 한 문장) + CTA `읽을 책 등록하기` + 서브카피 "책 사진 한 장·바코드로도 등록돼요". 기능 예고·게스트 배너는 **의미 단위 명시 개행**(#1144 — keep-all만으론 "둥지가/자라요" 구 찢김). 이유: 스토어 유입은 약속을 모른 채 이 화면을 만나며, 특히 재키(최대 차별점)가 등록 전엔 어디에도 노출되지 않았다. (구 카피 "아직 읽는 책이 없어요" 부정문 폐기)
- 슬라이드·튜토리얼 없음 — 위 카드 하나로 화면 추가 없이 어필(원칙 유지)
- CTA → **C** (책 등록)

### B. 사전 질문

**삭제.** 1일 1페이지 1문장이 유일한 목표.

### C. 첫 책 등록 (가입 전 가능)

**C-1. 검색 화면**

- 검색창: ISBN / 제목 / 저자
- 클라이언트 fuzzy 매칭 (Supabase 카탈로그/인라인 폴백 위에서. 한글 자모 분해 `es-hangul` + Fuse.js)
- 결과 카드: 표지 / 제목 / 저자 / 총 페이지
- 검색어 비어있을 때 하단 추천 탭: **요즘 Top 10** / **스테디 Top 10** (Phase 0 하드코딩 큐레이션)
- 결과 없을 시 "직접 등록" → C-2
- 우상단 "📚 내 서재" → 프로필 탭

**C-2. 확인 / 직접 등록 + 오늘의 한 문장 입력**

한 화면에서 책 등록 + 첫 기록 완료.

- 상단: 선택 책 표지·제목·저자 (직접 등록 시 제목·저자·총페이지 폼)
- "현재까지 읽은 페이지" 숫자 입력 (기본 0)
- **구분선**
- "오늘의 한 문장 🐦":
  - "어느 페이지에서?" 숫자 입력 (기본값 = 현재 페이지)
  - 텍스트 영역, placeholder "마음에 든 한 줄을 적어주세요 (최대 200자)"
  - 문자 수 카운터. 1자 이상이어야 CTA 활성
- 자동 임시 저장 (localStorage `rg_pending_sentence`)
- CTA `이 책으로 시작` → 이 책이 **활성 책** 지정 → **D**

### D. 세리머니 (성공 화면)

```
┌─────────────────────────────┐
│   [confetti 18조각, 8색]     │
│      🐦 (참새 bounce)        │
│      훌륭해요!                │
│   첫 한 문장을 남겼어요!       │
│  ┌─────┬─────┬─────┐        │   ← 보상 카드 3그리드
│  │ 🔥  │ ⚡  │ 🪵  │        │
│  │스트릭│+10XP│둥지+ │        │
│  └─────┴─────┴─────┘        │
│  [내일도 짹 →]               │
└─────────────────────────────┘
```

Confetti: 18조각, 8색 `#3FD17F #FFC233 #FF8A3D #5AB5F0 #F08A9A #B690F0 #2EB867 #FFD66B`, `duration 1.6~3s`, `delay 0~0.25s`, ~3.2초 후 정리, `z-index 110`.

- **게스트면 메인 CTA 아래 3차 텍스트 버튼 "이 기록, 계정에 저장하기" (#1134)** → 로그인 화면(구글/카카오/이메일) 진입. 첫 성공(aha) 직후가 저장 동기 최고점. 구 설계(`Google로 계속` OAuth 직행)는 카카오 주력 안드로이드와 안 맞아 로그인 화면 경유로 대체. 로그인 상태면 홈 복귀

### E. 가입

- 별도 랜딩 없음. D 세리머니 CTA가 OAuth 트리거
- OAuth 성공 후: localStorage 임시 데이터 → Supabase 동기화 ([backend.md §7.7](./backend.md)), 닉네임 미입력 시 E-1, 바로 **H**(홈)
- **계정 선택 강제 (#721)**: OAuth 요청은 항상 `prompt=select_account` 를 보낸다. 미지정 시 로그아웃(`signOut`=Supabase 세션만 정리) 후 재로그인하면 브라우저 잔류 Google 세션을 자동 선택해 **다른 계정으로 바꿀 수 없다**. 매번 계정 선택 화면을 띄워 계정 전환·추가를 보장.

> **게스트 우선 원칙 (v7.3, 2026-06-08)**: 로그인은 **벽이 아니라 저장 시점의 선택**이다.
> Supabase 모드에서도 **비로그인 사용자는 게스트로 앱 전체를 둘러볼 수 있다** (localStorage 백킹).
> **신규 게스트 = 빈 시작 (#1136, 2026-07-03 출시 결정)**: 구 데모 시드(데미안 102p·스트릭 12·XP 340·문장 2·완독 2·찜 3)는 Phase 0 시연용 잔재 — 스토어 유입이 "남의 기록"을 첫 화면에서 만나 신뢰를 깎는 문제로 제거. 신규 게스트는 §A 약속+기능 예고 카드(#1134)에서 시작한다. NPC(피드 타인 문장)는 의도된 소셜 기능이라 유지.
> **재방문 게스트 hydration (#1221)**: 빈 시작은 *진짜 신규*에게만 — 게스트가 책을 등록한 뒤 리로드하면 부팅 시 localStorage 활성 책·스트릭·XP를 appState 로 hydrate 해 홈이 그 책을 띄운다(로그인 buildStateFromSupabase 와 대칭). #1136 이후 이 경로가 없어 등록한 책이 리로드마다 "사라진" 것처럼 보이던 회귀 수정.
> 상단 게스트 배너(버튼 라벨 **"로그인"**, 흰 배경 2차 위계 — #1233: 구 solid '저장하기'는 진도 카드
> 저장하기와 동일 라벨·다른 의미로 오독 경로였고, 첫 화면 1차 solid 3개 경쟁을 만들었다)와
> 설정의 "🐦 로그인하고 내 기록 저장하기" 로 언제든 로그인 가능.
> 게스트가 첫 책·문장을 기록하면 `rg_v41.pending` 에 포착 → 로그인 직후 §7.7 동기화로 흡수.
> (구현: `app.js` 인증 게이트는 `showLogin` 온디맨드 + `syncPendingToSupabase`.)

**E-1. 닉네임 설정**
