## 4. 유저 시나리오 — 신규 가입 여정

각 단계는 화면 단위이며, **Phase 0 데모에서 모두 클릭으로 확인 가능해야 한다.**

### A. 진입 (비로그인)

- 헤더 슬로건 "하루 한 페이지, 한 문장에서 시작해요", 중앙 CTA `시작하기`
- 슬라이드·튜토리얼 없음
- CTA → **C** (책 등록)

### B. 사전 질문

**삭제.** 1일 1페이지 1문장이 유일한 목표.

### C. 첫 책 등록 (가입 전 가능)

**C-1. 검색 화면**

- 검색창: ISBN / 제목 / 저자
- 클라이언트 fuzzy 매칭 (`books.tsv` 위에서. 한글 자모 분해 `es-hangul` + Fuse.js)
- 결과 카드: 표지 / 제목 / 저자 / 총 페이지
- 검색어 비어있을 때 하단 추천 탭: **요즘 Top 10** / **스테디 Top 10** (Phase 0 하드코딩 큐레이션)
- 결과 없을 시 "직접 등록" → C-2
- 우상단 "📚 내 서재" → 프로필 탭

**C-1b. 📷 사진으로 시작 (OCR 첫 문장, #939 — cold start 신규 사용자 wedge)**

검색어가 비어있는 진입 상태(온보딩 첫 책)에 **검색창 타이핑 대신 사진 한 장으로 시작**하는 경로. 첫 책 등록의 가장 큰 마찰(검색·첫 문장 타이핑)을 제거한다. 근거: APPROVED `/office-hours` design doc(cold start — 온보딩 OCR 첫 문장, 2026-06-23).

- 진입: 추천 도서 위 **`📷 사진으로 시작`**(1차) + **`앨범`**(2차, 갤러리). 안내 "책 속 한 구절을 찍으면 문장과 책을 자동으로 찾아드려요".
- 흐름: 책 귀절 사진 → worker `/api/ocr`(Upstage Document OCR + solar-pro3 보정) 한 문장 추출 → ① 추출 문장을 **편집 가능 패널**(세리프, 최대 200자)에 채움(정확도 가드 — 틀리면 바로 수정) ② 같은 텍스트로 **책 추정**(검색 자동 트리거).
- 책 추정 성공: 후보에서 책 선택 → 책장 시트(읽는중) → 책 등록 + **추출·편집 문장을 첫 기록으로 저장**(타이핑 0). 게스트면 `pending`에 책+문장 포착(로그인 시 §7.7 흡수).
- 책 추정 실패(문장≠제목): "문장으론 책을 못 찾았어요" + **`제목으로 검색`**(검색어만 비우고 문장 패널은 보존 — 재타이핑 없음) 으로 **수동 검색 폴백**.
- **게스트 OCR 가능**: `ocrProxy`(`worker/index.mjs`)는 동일출처 `Origin` + 서버 `UPSTAGE_API_KEY` 만 요구, 로그인·토큰 게이트 없음 → 가입 전 게스트도 호출. (읽기모드 빠른입력 OCR #498 과 **공유 헬퍼 `window.ocrExtractSentence`** 단일화 — 중복 구현 없음.)
- 구현: `SearchModal`(`js/search.js`) 진입점·OCR 패널, `handleSearchSelectBook(book, shelf, { firstSentence })`(`js/app.js`) 첫 문장 영속, `ocrExtractSentence`(`js/data.js`) 공유 호출.

**C-2. 확인 / 직접 등록 + 오늘의 한 문장 입력**

한 화면에서 책 등록 + 첫 기록 완료.

- 상단: 선택 책 표지·제목·저자 (직접 등록 시 제목·저자·총페이지 폼)
- "현재까지 읽은 페이지" 숫자 입력 (기본 0)
- **구분선**
- "오늘의 한 문장 🐦":
  - "어느 페이지에서?" 숫자 입력 (기본값 = 현재 페이지)
  - 텍스트 영역, placeholder "마음에 든 한 줄을 적어주세요 (최대 200자)"
  - 문자 수 카운터. 1자 이상이어야 CTA 활성
- 자동 임시 저장 (localStorage `rg_pending_sentence`)
- CTA `이 책으로 시작` → 이 책이 **활성 책** 지정 → **D**

### D. 세리머니 (성공 화면)

- 가입 전이면 CTA `Google로 계속` (OAuth 직행), 로그인 상태면 홈 복귀

### E. 가입

- 별도 랜딩 없음. D 세리머니 CTA가 OAuth 트리거
- OAuth 성공 후: localStorage 임시 데이터 → Supabase 동기화 ([backend.md §7.7](./backend.md)), 닉네임 미입력 시 E-1, 바로 **H**(홈)
- **계정 선택 강제 (#721)**: OAuth 요청은 항상 `prompt=select_account` 를 보낸다. 미지정 시 로그아웃(`signOut`=Supabase 세션만 정리) 후 재로그인하면 브라우저 잔류 Google 세션을 자동 선택해 **다른 계정으로 바꿀 수 없다**. 매번 계정 선택 화면을 띄워 계정 전환·추가를 보장.

> **게스트 우선 원칙 (v7.3, 2026-06-08)**: 로그인은 **벽이 아니라 저장 시점의 선택**이다.
> Supabase 모드에서도 **비로그인 사용자는 게스트로 앱 전체를 둘러볼 수 있다** (localStorage 백킹).
> 상단 게스트 배너 + 설정의 "🐦 로그인하고 내 기록 저장하기" 로 언제든 로그인 가능.
> 게스트가 첫 책·문장을 기록하면 `rg_v41.pending` 에 포착 → 로그인 직후 §7.7 동기화로 흡수.
> (구현: `app.js` 인증 게이트는 `showLogin` 온디맨드 + `syncPendingToSupabase`.)
