# 한 문장 외부 공유 카드 — "내 한 문장을 밖으로"

> **신규 (v8.5, 2026-06-16)**: 외부 공유 기능 신설(에픽 #650)의 A안 — 한 문장 외부 포스팅 카드 디자인. 근거: 북모리 이미지 카드 공유 = 핵심 바이럴 루프(`meta/decisions.md` v5.1 "모이 → 이미지 카드 공유 채택"). 이슈 #651.
> **편집 정책**: 이 영역(외부 공유 카드 비주얼·포맷) 변경은 이 파일 PR로. spec-only PR 준수. 구현(canvas/DOM-to-image·Web Share API)은 후속 코드 PR.

## 1. 목표 (What & Why)

> **"읽다가 끌어올려진 한 문장을, 그대로 인스타·카톡·X에 올릴 수 있게."**

- **What**: 내가 저장한 한 문장(인용/감상)을 앱 밖으로 내보내는 **이미지 공유 카드**의 비주얼·포맷·토큰 정의. 진입은 책 상세 모달의 한 문장 카드·피드 카드(#650 A) — 진입 동선·기술 스택은 본 spec의 범위가 아니다(§8 비목표).
- **Why (바이럴 루프)**: 카드 하단 워터마크에 서비스 링크/핸들을 실어 **밖에서 본 사람 → 유입**으로 잇는다. 북모리가 검증한 핵심 루프(`meta/decisions.md` v5.1)를 ReadingGo 톤으로 재현.
- **코어 감정**: 고양감(uplift) — 카드는 "오늘 나를 끌어올린 한 문장"을 박제하는 작은 트로피처럼 느껴져야 한다. 정보 나열이 아니라 **한 문장이 주역**인 여백·타이포.
- **SLC**: 한 장의 카드를 'small but delightful'하게. 인용/감상 2종 × 1:1/9:16 2비율 + 텍스트 폴백 — 그 이상은 후속(§8).

## 2. 카드 구성 (Composition)

위→아래 4영역. 비율이 바뀌어도 **순서·역할은 동일**, 여백만 재배분.

| 영역 | 내용 | 역할 |
|---|---|---|
| ① 종류 칩 | `❝ 인용` 또는 `💭 감상` | 이 카드가 작품 문장인지 내 생각인지 한눈에 |
| ② 문장 (주역) | 저장한 한 문장 | **시각 무게의 최소 60%**. 가장 큰 타이포·최대 여백 |
| ③ 출처 | 책 표지(소) + 제목 + 저자 | 어느 책의 문장인지. 표지는 작게, 문장을 압도하지 않음 |
| ④ 워터마크 | ReadingGo 로고 + 핸들/링크 | 바이럴 진입점. 항상 노출(끌 수 없음) |

- **위계**: ② > ③ > ④ > ① 순으로 크다. 종류 칩(①)은 작지만 색으로 구분(§5).
- **문장 길이 대응**: 한 문장은 짧음(앱이 한 문장 강제, `sentences.text`). 글자 수에 따라 ②의 폰트 크기를 단계적으로 조정(§6 clamp). 아주 길면 줄바꿈 허용, 그래도 넘치면 말줄임(`…`) 후 "전문은 앱에서" 미세 라벨.
- **빈 표지 대응**: `cover_url` 없으면 표지 자리에 브랜드 틴트 사각형 + 제목 이니셜(피드 기본 아바타 규칙 재사용).

## 3. 레이아웃 — 1:1 (피드)

인스타 피드·카톡 프로필·X 인라인. 정사각. 문장이 정중앙 시상대.

```
┌───────────────────────────────┐
│  ❝ 인용                         │  ① 좌상단 종류 칩
│                                 │
│                                 │
│    오래 보아야                   │  ② 문장 — 중앙 정렬, 최대 타이포
│    사랑스럽다.                   │     상하 여백 넉넉 (시상대)
│    너도 그렇다.                  │
│                                 │
│                                 │
│  ┌──┐                           │
│  │표 │  풀꽃                     │  ③ 출처 — 좌하단, 표지(소)+제목/저자
│  │지 │  나태주                   │
│  └──┘                           │
│ ─────────────────────────────  │     얇은 구분선(--line)
│  🐦 ReadingGo   @readinggo.app  │  ④ 워터마크 — 로고 좌 / 핸들 우
└───────────────────────────────┘
   1080 × 1080 (1:1)
```

- 캔버스 패딩: 카드 폭의 ~8%(`--pad-x`). 배경 `--paper`.
- ② 문장은 카드 세로 중앙대(약 35~65% 구간)에 앉힌다. ❝/❞ 따옴표 글리프는 인용일 때만 장식으로 옅게(§5).
- ③ 표지 높이 = 카드의 ~12%. 제목 Rounded, 저자 `--ink-2` 한 단계 작게.

## 4. 레이아웃 — 9:16 (스토리) — 후속/미구현 (spec-ahead)

> 📌 **미구현 · spec-ahead (드리프트 정정 2026-07-09)**: 현재 `share-card.js`는 **1:1(1080×1080)만** 생성한다. 아래 9:16 스토리 레이아웃은 **후속(#650 B 동반)** 이며 코드 미반영(코드 주석도 "9:16 스토리는 후속"). §1 SLC의 "2비율"은 목표치일 뿐 현행 구현 아님.

인스타/카톡 스토리. 세로로 길다 → 문장이 위쪽 황금비 지점, 출처·워터마크는 아래로 충분히 내림(스토리 UI 가림 영역 회피).

```
┌─────────────────────┐
│                      │
│  ❝ 인용               │  ① 상단
│                      │
│                      │
│   오래 보아야         │  ② 문장 — 상단 1/3~중앙
│   사랑스럽다.         │     세로 여백 가장 크게
│   너도 그렇다.        │
│                      │
│                      │
│                      │
│                      │
│  ┌──┐                │
│  │표 │ 풀꽃           │  ③ 출처 — 하단부
│  │지 │ 나태주         │
│  └──┘                │
│ ──────────────────   │
│  🐦 ReadingGo         │  ④ 워터마크 — 최하단
│     @readinggo.app    │     (스토리 하단 UI 위 안전영역)
└─────────────────────┘
    1080 × 1920 (9:16)
```

- ④ 워터마크는 바닥에서 ~10% 띄운다(스토리 답장창·UI 겹침 방지 = 세이프 영역).
- 문장이 짧으면 ②를 더 위로(상단 1/3), 길면 중앙까지 자연 확장.
- ①②③④ 좌측 정렬 기조(피드는 ②만 중앙). 세로 카드에서 중앙정렬 긴 문장은 읽기 흐름이 끊겨 좌측 정렬이 안정적.

## 5. 인용 vs 감상(💭) 구분

> **⚠️ #596 폐기**: '내 생각'(thought)은 멸종([nest.md §147](./nest.md)) — 실제 카드는 **항상 인용(quote)**. 아래 감상(💭) 표는 `sentences.kind` 컬럼 롤백 안전상 남은 *비활성* 명세(현재 thought 행 0). '내 생각'은 문장에 단 `my_note`로 표현하고, 카드도 그 인용을 보여준다. (office-hours 2026-06-24 재확인.)

구 `sentences.kind`(`quote`|`thought`, companion §2 #359/#360)를 카드에 반영하던 명세. **색·칩·장식**으로 구분 — 현재는 인용(quote)만 노출.

| 구분 | 종류 칩 | 강조색 | 장식 | 의미 |
|---|---|---|---|---|
| **인용 (quote)** | `❝ 인용` | `--brand`(민트) | 큰 따옴표 글리프 `❝ ❞` 옅게(`--brand-soft`) | 작품 속 문장 — 책이 한 말 |
| **감상 (thought)** | `💭 감상` | `--violet`(보라) | 따옴표 없음, 문장 앞 작은 `💭` | 내가 한 생각 |

- 인용은 "책의 목소리" → 브랜드 민트 + 따옴표로 *인용임*을 시각화. 감상은 "내 목소리" → 보라 틴트로 결을 달리해 인용과 헷갈리지 않게(companion이 LLM 차원에서 지키는 구분을 카드에서도 유지).
- 감상 카드는 출처(③)에 책을 **"~을 읽고"** 뉘앙스로 (예: `풀꽃 · 나태주를 읽고`) — 문장이 작품 인용이 아니라 그 책에 대한 내 생각임을 명확히.

## 6. 적용 토큰 (Design Tokens)

기존 브랜드 토큰(`index.html` `:root`, design.md §11) **재사용**. 신규 토큰 도입 없음 — 카드 전용 치수만 로컬 정의.

### 6.1 색

| 용도 | 토큰 |
|---|---|
| 카드 배경 | `--paper` (#FAF6F0) |
| 문장 본문 | `--ink` (#2A2D33) |
| 출처 제목 | `--ink` / 저자 `--ink-2` |
| 구분선 | `--line` (#ECE6DA) |
| 인용 강조·칩 | `--brand` / 칩 배경 `--brand-soft` |
| 감상 강조·칩 | `--violet` / 칩 배경 `--violet-soft` |
| 워터마크 핸들 | `--ink-3` (#9097A0) |

### 6.2 타이포 (design.md §11.1 역할 분리 준수)

| 요소 | 폰트 | 비고 |
|---|---|---|
| 문장(②) | Moneygraphy Rounded | `letter-spacing:-0.2px`, `line-height:1.45` |
| 출처 제목·저자 | Moneygraphy Rounded | 저자 한 단계 작게 |
| 종류 칩·워터마크 핸들 | Moneygraphy Pixel | 메타·라벨 → Pixel, `letter-spacing:.5px` |

- **문장 폰트 크기 = 글자 수 반응형**: 1:1 기준 짧음 약 `clamp` 상한, 길수록 단계 축소(가독 하한까지). 한 문장이라 보통 2~4줄 안에 든다. Pixel은 **굵기 변경 불가**(단일 굵기) — 칩/핸들 등 짧은 라벨에만.

### 6.3 여백·모양

- 카드 라운드(미리보기 UI): `--r-lg`(22px). 내보낸 이미지 자체는 SNS가 모서리를 깎으므로 안전영역만 확보.
- 영역 간 간격: 8의 배수 스케일(앱 기조와 동일). 문장 영역은 의도적으로 과한 상하 여백(시상대 느낌).
- 그림자는 **미리보기 카드에만**(`--shadow-1`), 내보낸 이미지에는 넣지 않음(배경 투명/흰 위에서 어색).

## 7. 텍스트-only 폴백 (이미지 미지원·접근성)

이미지 생성 실패·미지원 환경(일부 X 클라, 텍스트 붙여넣기, 스크린리더)에서의 **순수 텍스트 포맷**. Web Share API의 `text`/클립보드로 쓰인다(기술은 후속, 포맷만 규정).

```
"오래 보아야 사랑스럽다. 너도 그렇다."
— 나태주, 《풀꽃》

📖 ReadingGo에서 내 한 문장 남기기
https://readinggo.hyuniverse.workers.dev
```

- **인용**: 문장을 따옴표로 감싸고 `— 저자, 《책제목》`.
- **감상(💭)**: 따옴표 대신 `💭` 프리픽스 + `(《책제목》을 읽고)` 꼬리.
  ```
  💭 오래 본다는 건 결국 시간을 들이는 일.
  (나태주 《풀꽃》을 읽고)

  📖 ReadingGo에서 내 한 문장 남기기
  https://readinggo.hyuniverse.workers.dev
  ```
- 마지막 2줄(서비스 한 줄 + 링크)은 **항상 포함** — 텍스트 경로에서도 바이럴 링크 유지.
- 접근성: 이미지 카드 생성 시 `alt`는 폴백 텍스트 본문(문장+출처)을 그대로 사용(워터마크 링크 제외).

## 8. 비목표 / 후속 (Non-goals)

- **진입 동선·버튼 UI**: 책 상세/피드 어디에 공유 버튼을 두는지·인터랙션은 #650 본체와 해당 피처 spec(feed/profile) 소관. 본 spec은 카드 *결과물*만.
- **기술 구현**: Web Share API·클립보드 폴백·이미지 생성 라이브러리 선택은 후속 코드 PR(§9 구현 메모).
- **B안(서비스 자체 공유·referral)**: #652 별도 디자인.
- **마일스톤 공유 카드**(50·100일 스트릭): `meta/decisions.md` v6 채택분, 후속. 본 카드 골격을 재사용하되 별도 디자인.
- **여러 문장·대화(짹 Q/A) 공유**: 한 문장 1장 집중 유지. 묶음 공유는 후속 검토(open-issues).
- **사용자 테마/배경 선택, 폰트 임베드 라이선스 점검**: 후속.

## 9. 구현 메모 (Stack Lock 주의 — 비결정)

> 아래는 구현 시 검토 항목이며 **본 spec PR로 결정하지 않는다**. 라이브러리 도입은 Stack Lock 룰에 따라 별도 확인 필요.

- **이미지 생성**: 두 갈래 — ① `<canvas>` 직접 렌더(의존성 0, 폰트·줄바꿈 수작업) ② DOM-to-image류(`html-to-image` 등, 마크업 재사용 쉬움·**라이브러리 도입 승인 필요**). SLC·의존성 최소 관점에선 canvas가 1순위 후보지만, 한글 폰트 메트릭·이모지 렌더 리스크가 있어 시안 단계에서 양쪽 PoC 권장.
- **폰트**: 카드에 Moneygraphy 임베드 시 OTF 서브셋·라이선스 확인. canvas 렌더는 폰트 로드 완료(`document.fonts.ready`) 후 그려야 글리프 누락 없음.
- **링크/핸들 값**: 워터마크 URL은 현 데모(`readinggo.hyuniverse.workers.dev`) 기준. 정식 도메인·핸들 확정 시 한 곳에서 치환되도록 상수화.
- **시안 확인**: 본 spec과 함께 제출된 프로토타입 [`prototypes/share-card.html`](../prototypes/share-card.html) 에서 1:1·9:16 × 인용·감상 + 텍스트 폴백을 눈으로 확인.
