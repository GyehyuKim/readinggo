## 1. 범위 (What's This)

- **대상**: ReadingGo **서비스 자체**를 외부(카톡·인스타·X·문자 등)로 공유하는 동선. 지인 추천(친구 초대)과 광고성 유입의 진입 표면을 설계한다.
- **이 문서가 다루는 것**: 공유 페이로드 구성, 진입점, 랜딩(공유 링크 클릭 첫 화면)·전환 흐름, referral 정책 **초안**(보상은 TBD).
- **이 문서가 다루지 않는 것 (분리)**:
  - **한 문장 카드 공유**(인용/감상을 이미지 카드로) = #651, [share.md](./share.md). 본 문서는 *서비스* 공유이고, share.md는 *콘텐츠*(한 문장) 공유다. **비주얼은 공유**(§6) 하되 동선·목적이 다르다.
  - referral 백엔드(코드 발급·귀속/attribution·중복방지) 구현 = Phase 1+ Supabase. 본 문서는 **스펙 레벨 기술만** 하고 구현하지 않는다.

> **한 줄**: "이 앱, 너도 써봐" 를 **마찰 없이 한 번에** 보내고, 받은 사람이 **무엇인지 3초 안에** 이해하고 시작하게 한다.

## 2. 공유 페이로드 (Share Payload)

서비스 공유 메시지는 **철학 한 줄 + 링크** 두 조각이다. SLC — 길지 않게, 한 호흡에 읽히게. 특정 책 인용은 넣지 않는다(#1092).

| 조각 | 내용 | 예시 |
|---|---|---|
| **서비스 소개(철학)** | 우리 철학 한 줄 — 사람들이 책을 더 쉽게·더 깊게 읽게 한다는 가치. 자랑이 아니라 권유. ("Duolingo" 비유 금지 — [README §0](./README.md)) | "책을 더 쉽게, 더 깊게 읽도록. 하루 한 문장으로 독서 습관을 만드는 ReadingGo." |
| **방문/설치 링크** | referral 파라미터가 붙은 서비스 링크 | `https://readinggo.app/?ref=ABC123` |

- **특정 책 인용은 미동반(#1092)**: ~~대표 한 문장(공유자의 최근 인용)~~ 은 빼고 철학 한 줄만 보낸다. 앱 초대의 목적은 책 한 권 공유가 아니라 "이 습관을 같이"이기 때문(베타 피드백 — "특정 책의 문구를 공유하는 게 목적이 아님"). 한 문장 *카드* 공유(#651/[share.md](./share.md))는 별개 동선으로 유지.
- **미리보기 = 링크 OG 카드**: 공유 텍스트를 늘리는 대신 랜딩(`index.html`)의 Open Graph 메타(`og:title`·`og:description`·`twitter:card`)로 카톡·아이메시지·인스타에 미리보기 카드를 띄운다. `og:description` = 위 철학 한 줄과 동일(공유 텍스트와 한목소리). `og:image`(1200×630)는 미정 — 현재 텍스트 카드(`summary`), 추후 추가 시 `summary_large_image`.
- **공유 수단**: Web Share API(`navigator.share`) 우선, 미지원 환경은 **클립보드 복사 + 토스트** 폴백(#650-A와 동일 기술 스택, 프레임워크 추가 없음).
- **이미지 카드 vs 텍스트+링크**: 서비스 공유의 기본은 **텍스트+링크**(전환 링크가 1순위). 이미지 카드가 필요하면 #651의 한 문장 카드 비주얼을 재사용하되 **하단 워터마크 영역에 "→ ReadingGo 시작하기" CTA + 링크**를 덧댄 변형으로 한다(§6).

### 2.1 카피 톤

- 자랑이 아니라 **권유**. "나 이만큼 읽었어"가 아니라 "이거 같이 해볼래?".
- 1인칭·구어체 1~2문장. 광고 문구처럼 들리지 않게(친구가 보낸 느낌 유지).
- 마이크로카피 수치·확정 문구는 [design.md](./design.md)(owner: seungwon) 후속 PR에서 토큰화 — 본 문서는 방향만.

## 3. 진입점 (Entry Points)

서비스 공유는 **자발적 진입 + 모멘트 유도** 두 갈래.

| # | 진입점 | 위치 | 트리거 | 비고 |
|---|---|---|---|---|
| E1 | **프로필 공유 버튼** | 프로필 탭 헤더 (내 책장/통계 옆) | 상시(사용자가 누름) | 자발적. 가장 기본. **⚠️ 미구현 (드리프트 정정 2026-07-09)** — 현재 E2 서재 공유만 존재, 프로필 헤더 진입은 Phase-0 통합(#727) 후속 |
| E2 | **서재 공유 버튼** | 내서재 화면 액션 영역 | 상시 | 내가 읽은 책 큐레이션을 "이런 거 읽어" 맥락으로 권유 |
| E3 | **온보딩 완료 유도** | 가입 여정 마지막 단계(E) | 첫 체크인/온보딩 완료 직후 1회 | 가벼운 "친구도 초대할래요?" 카드. **스킵 가능·페널티 없음** |
| E4 | **마일스톤 달성 유도** | 성(🏰) 획득·완독·스트릭 분기점 | 성취 세리머니 직후 | 고양감이 가장 높은 순간에 권유. 강요 X, 1회성 |

- **모멘트 유도(E3·E4)는 인터럽트가 아니라 제안**: 세리머니/완료 화면 *안*에 작은 칩/배너로 얹는다. 별도 모달 폭격 금지([companion.md](./companion.md) §3 "별도 탭 신설 안 함" 정신 계승).
- **빈도 제어**: E3는 가입당 1회. E4는 같은 종류 성취당 **쿨다운**(예: 직전 노출 후 N일·M회 성취 시 1회 — 수치 TBD, 어뷰징·피로 방지). 사용자가 닫으면 그 모멘트는 다시 안 띄움.
- **자발적 진입(E1·E2)** 은 상시 노출하되 시각적으로 과하지 않게(아이콘+라벨 수준).

### 3.1 공유 시트 (Share Sheet)

진입점을 누르면 뜨는 공통 시트(한 컴포넌트). 진입점별로 프리필만 다름.

```
┌──────────────────────────────────────┐
│  친구에게 ReadingGo 권하기          ✕ │
├──────────────────────────────────────┤
│  ┌──────────────────────────────┐    │
│  │ "끝내 닿지 못해도 향하는      │    │  ← 대표 한 문장 미리보기
│  │  마음에 대하여."             │    │    (없으면 이 박스 생략)
│  │  — 우리가 빛의 속도로…        │    │
│  └──────────────────────────────┘    │
│                                       │
│  하루 한 문장으로 쌓는 독서 습관,     │  ← 서비스 소개 한 줄
│  ReadingGo.                           │
│                                       │
│  🔗 readinggo.app/?ref=ABC123         │  ← 내 추천 링크(복사 가능)
│                                       │
│  [ 공유하기 ]   [ 링크 복사 ]         │  ← navigator.share / 클립보드
│                                       │
│  내 추천 코드: ABC123                 │  ← referral 코드 (§4)
│  (보상 안내 영역 — TBD, §4.3)        │
```

- 한 컴포넌트(`ServiceShareSheet` 가칭)로 E1~E4 공통. 진입점은 `source`(profile/library/onboarding/milestone) 만 전달 → 분석 이벤트·프리필 분기.

## 4. referral 요소 (DRAFT)

> ⚠️ 본 절의 **보상 관련 결정은 전부 TBD**. 코드/비용·Phase 영향이 있어 **CEO 검토 후 확정**(§8 오픈 질문). 보상 미확정 상태에서도 **"추천 코드·링크 + 귀속 카운트"** 까지는 가치가 있으므로(친구가 누구 통해 왔는지) 그 부분부터 도입 가능.

### 4.1 구성 요소

| 요소 | 설명 | Phase |
|---|---|---|
| **추천 코드** | 사용자별 고유 단축 코드(예: `ABC123`). 사람이 읽고 말할 수 있는 길이 | Phase 1 |
| **추천 링크** | 코드를 쿼리로 실은 서비스 링크(`?ref=CODE`) | Phase 1 |
| **귀속(attribution)** | 신규 가입자가 어떤 코드로 들어왔는지 기록(랜딩에서 코드 캡처 → 가입 시 연결) | Phase 1 |
| **보상** | 추천인/피추천인에게 줄 보상(유무·형태) | **TBD — CEO 검토** |

- 추천 코드/링크·귀속은 **Supabase(Phase 1+)** 에 둔다. 게스트(Phase 0)는 코드 발급 없이 **서비스 소개+링크만** 공유(referral 비활성, graceful degrade).
- 코드 생성·중복방지·셀프리퍼럴/어뷰징 차단은 **백엔드 스펙**([backend.md](./backend.md)) 후속 — 본 문서는 표면만.

### 4.2 referral 보상 모델 — 옵션 비교 (택1 전 검토용)

> 아래는 **제안 옵션**이며 어느 것도 확정 아님. 코어(고양감·읽기 습관)와 어긋나는 보상(예: 현금성)은 지양.

| 모델 | 추천인 보상 | 피추천인 보상 | 장점 | 단점·리스크 |
|---|---|---|---|---|
| **A. 무보상(코드만)** | 없음(귀속 카운트만 노출 — "N명이 나로 시작했어요") | 없음 | 비용 0, 가장 단순, 즉시 도입, 어뷰징 무의미 | 외재적 유인 약함(바이럴 계수↓) |
| **B. 인앱 비현금 보상** | XP·뱃지·둥지 가속 등 인앱 재화 | 동일(소액 XP·시작 부스트) | 코어와 정렬(읽기 동기), 현금비용 0 | XP 인플레·게임 밸런스 영향([systems.md](./systems.md) 소유 승원과 조율 필요), 어뷰징 방지 필요 |
| **C. 기능 언락** | 추천 N명 시 프리미엄 기능 일시 개방(예: 짹 무료턴 상향) | 동일 | 수익화와 연결, 진짜 가치 전달 | 프리미엄 정의 선행 필요(수익 모델 미정 — [meta/open-issues.md](./meta/open-issues.md)) |
| **D. 현금성/실물** | 기프티콘·할인 등 | 동일 | 강한 유인 | **비용·정산·법적(전자상거래/세무) 부담 큼**, Phase 0/1 범위 초과 — **비권장** |

- **초안 권고(비확정)**: **A로 시작**(보상 없이 코드+귀속만) → 데이터 보고 필요 시 **B(인앱 비현금)** 검토. **D는 학기 범위 밖**. 단 최종 선택은 §8 CEO 검토 결과를 따른다.

### 4.3 보상 표시 영역

- 공유 시트(§3.1)·랜딩(§5)에 **보상 안내 자리**를 미리 비워둔다(레이아웃 슬롯). 보상 미확정 동안은 **숨김** 또는 중립 문구("함께 읽을 친구를 초대해요")만 노출.
- 보상이 확정되면 이 슬롯에 추천인/피추천인 보상 문구를 채운다 — **레이아웃 재설계 없이** 텍스트만 주입(향후 변경 비용↓).

## 5. 랜딩 — 공유 링크 클릭 첫 화면 (Conversion)

추천 링크(`?ref=CODE`)로 들어온 **비로그인/신규** 방문자가 처음 보는 화면. 목표 = **3초 이해 → 시작**.

```
┌──────────────────────────────────────┐
│            🐦 ReadingGo               │
│                                       │
│   "끝내 닿지 못해도                    │  ← 추천인의 대표 한 문장
│    향하는 마음에 대하여."             │    (있으면. 사람 냄새 = 전환↑)
│                                       │
│   ○○님이 ReadingGo로 초대했어요.      │  ← 추천인 맥락(닉네임, 코드 유효 시)
│                                       │
│   하루 한 문장으로 쌓는 독서 습관.    │  ← 서비스 가치 한 줄
│                                       │
│   ┌────────────────────────────┐     │
│   │     시작하기 (가입)         │     │  ← 1순위 CTA
│   └────────────────────────────┘     │
│   먼저 둘러보기 →                     │  ← 2순위(게스트 체험)
│                                       │
│   (보상 안내 슬롯 — TBD, §4.3)       │
└──────────────────────────────────────┘
```

### 5.1 전환 흐름

```
추천 링크 클릭
   ↓
랜딩(위) — ref 코드를 클라에 임시 보관(쿼리/스토리지)
   ↓
[시작하기] → 온보딩/로그인 → 가입 완료 시 ref 코드를 가입자에 귀속(§4.1)
   │
   └ [먼저 둘러보기] → 게스트 체험 → 추후 가입 시에도 보관된 ref 코드 귀속(만료 정책 TBD)
```

- **ref 코드 보관**: 클릭~가입 사이 이탈 대비, 코드를 임시 보관(만료·우선순위 규칙은 backend 후속). 직접 방문(코드 없음)도 정상 동작(랜딩 = 일반 소개 화면으로 graceful).
- **유효하지 않은/만료 코드**: 추천인 맥락 줄만 숨기고 일반 소개 랜딩으로 폴백(에러 화면 금지).
- **자기 자신 링크**: 이미 로그인한 사용자가 자기 링크로 오면 셀프리퍼럴 방지 — 귀속하지 않고 홈으로.

### 5.2 인앱 브라우저 탈출 (#1096)

카톡·인스타·페북 등으로 공유된 링크는 **인앱 브라우저(WebView)** 로 열려 전환을 막는다:
- 외부 Chrome/Safari 와 세션 분리 → 항상 비로그인(참여·가입하려면 로그인 필요)
- **Google 로그인 차단** — Google 이 embedded WebView OAuth 를 거부(`disallowed_useragent`, 2021-09-30~). 카카오 로그인만 가능
- 링크 복사 막힘 → 외부로 옮길 수 없음

**대응**: 부팅 시 인앱 브라우저 감지(`navigator.userAgent` 의 `KAKAOTALK`·`Instagram`·`FBAN` 등 — `inapp.js` `window.RG_inApp.detect()`) → 상단 배너로 "기본 브라우저로 열기" 유도. 카카오는 `openExternal` 스킴(iOS `kakaotalk://web/openExternal?url=`, Android `intent://…package=com.android.chrome`)으로 원클릭, 그 외(인스타·페북)는 "⋯ → 다른 브라우저로 열기" 안내만(공식 스킴 없음). 외부로 빠지면 로그인·복사·공유가 정상화된다. 초대 링크 `?r=`([co-reading.md §5.2](./co-reading.md))·데모 링크 공유 전반에 공통 적용.
