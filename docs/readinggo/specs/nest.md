### 5.1 홈 탭 — 화면 구성

홈 탭(하단 탭 라벨 '홈', 내부 탭 키 `nest`)은 **책읽기 중심**으로 구성된다(v8, #432·#436). 활성 책 1권의 진척 배너 아래 읽기 세션을 인라인으로 펼치고, 오늘 남긴 한 문장을 바로 보여준다. (구 The Path 경로 시각화는 v7에서 폐기. **둥지 캐릭터(NestTheatre)는 v8부터 홈이 아닌 [profile.md §5.8](./profile.md) 최상단**에서 렌더된다.)

### 5.2 둥지 진화 (Dynamic Stage) — 1,600 XP 주기

> **v8.1 변경 (2026-06-15, #520)**: 전체 누적 XP를 그대로 단계에 매핑하는 모델을 대체한다.
> 둥지는 책과 분리된 XP 기반 시각화라는 원칙은 유지하되, **1,600 XP마다 성을 완성하고 새 둥지 주기를 시작**한다.

XP 값은 차감 없이 계속 누적한다([systems.md §6.3](./systems.md)). 둥지 표시만 현재 주기 값으로 계산한다.

```text
cycleXp     = totalXp % 1600
castleCount = floor(totalXp / 1600)
```

| 단계 | 현재 주기 XP | 이모지 | 단계명 | 색상 | 배경색 |
|---|---:|---|---|---|---|
| 1 (시작) | 0–99 | 🪵 | 나뭇가지 자리 | #AFAFAF | #f3f4f6 |
| 2 | 100–399 | 🪹 | 빈 둥지 | #F59E0B | #FEF3C7 |
| 3 | 400–899 | 🏠 | 따뜻한 둥지 | #58CC02 | #F0FDF4 |
| 4 | 900–1,599 | 🏡 | 다정한 집 | #1CB0F6 | #EFF6FF |
| 5 (경계 연출) | 1,600 도달 순간 | 🏰 | 참새의 성 | #CE82FF | #FAF5FF |

- Lv5는 **1,600 XP 달성 시 도달하는 주기 완료 단계**다. 성 획득 후 현재 주기 XP는 `0`이 되고 둥지는 Lv1부터 다시 시작한다.
- 책을 바꾸거나 완독해도 `totalXp`와 현재 주기는 유지된다. 완독 자체는 성 획득 트리거가 아니다.

진화 시 마이크로카피:

| 전환 | 카피 |
|---|---|
| LV1→2 | "참새가 자리를 잡았어요!" |
| LV2→3 | "참새가 살림을 차렸어요!" |
| LV3→4 | "다정한 이웃이 되었어요!" |
| LV4→5(주기 완료) | "전설의 참새 성주!" |

- XP가 쌓이면 현재 주기 안에서 단계가 상승하고, 1,600 XP 경계를 넘으면 성(🏰)을 획득한다(§5.2.1).
- v7 폐기: ~~첫 7일 D1/D3/D7 가속~~, ~~운영자 졸업 짹 동시 트리거~~.

#### 5.2.1 성(🏰) — XP 주기 완료 지표

- **성 개수 = `floor(totalXp / 1600)`.** 별도 DB 컬럼이나 테이블 없이 `users.xp`에서 파생한다([backend.md §7.2/§7.3](./backend.md)).
- **획득 시점 = XP 추가 후 1,600 경계 통과.** Lv5 도달 세리머니 후 다음 주기를 시작한다.
- 완독 책은 [profile.md §5.8](./profile.md)의 읽은 책 목록에 그대로 남지만, 더 이상 성 개수나 성 카드로 환산하지 않는다.
- XP 차감이 없으므로 획득한 성은 줄어들지 않는다.

### 5.3 활성 책 전환

여러 책을 동시에 읽을 수 있으나, **항상 한 권이 "활성 책"**.

- 활성 책 = 홈 탭이 그리는 책
- **현재 구현 = 좌우 리볼빙 캐러셀 (#185, v7.2)**: 홈 책 카드에 `‹ ›` 화살표 + 하단 점 인디케이터. 읽는 중 책이 2권 이상이면 노출, 좌우로 넘겨 **즉시 활성 책 전환**. 전환 = `window.RG_activateBook` → `activeBook.set` + 둥지 재계산
- **각 책의 세션·진척·한 문장은 독립 보존**(책 전환 시 데이터 초기화 금지)
- 데이터: `users.active_user_book_id` ([backend.md §7.8](./backend.md)). 전환 = `DataStore.activeBook.set`
- ~~구 설계: 표지/제목 탭 → 전환 **시트** 슬라이드업~~ → **캐러셀로 대체(시트 폐기, v7.2)**. 별도 `ActiveBookSheet` 컴포넌트 없음.

### 5.4 일일 미션 — [§4](./onboarding.md) D 참조

- 하루 1세션. 같은 날 재진입 시 "오늘은 이미 완료했어요" + 한 문장 추가 옵션(서브 액션, 스트릭·XP 영향 없음)
- 입력: 현재 페이지 + 한 문장(페이지 강제·문장 선택, **1000자 이내**). **그 문장이 속한 페이지로 기록**(진행률과 별개 개념 — "📍 N쪽의 문장")
- 완료 시 즉시 보상: XP +20 + 스트릭 +1 + 둥지 진척 갱신 ([systems.md](./systems.md))
- **체크인 세리머니 (v8 갱신, #449/#426)**: 스트릭 카드·"N일 연속" 문구 **제거** — 대신 **누적 XP 기반 둥지 진척도 bar**(체크인 전→후 애니메이션)로 "오늘 한 쪽이 둥지를 키운다"를 보여줌. 둥지 단계가 오르면(`nestUp`) 별도 step의 **진화 축하 화면**(이전 단계 → 다음 단계 + 카피 + "계속 →")으로 전환
- **한 문장 보상 카드 정직 표시 (#549)**: 세리머니 보상 그리드의 한 문장 카드는 **문장을 실제로 남겼을 때만 `🔖 저장됨`**. 페이지만 업데이트(문장 미입력)했는데 "저장됨"이 뜨던 거짓 표시 제거 — 미입력 시 **이 책에서 모은 한 문장 수(`📖 N개 · 이 책 한 문장`)**, 0개면 **`✍️ 한 문장 남겨봐요` 독려**로 표시
- **완독 시(100%)**: 완독 세리머니 + 완독 배지 + Confetti(주기 완료 세리머니와 별개) → 별점·소감 입력([§5.8](./profile.md)). 완독 보상 XP가 1,600 경계를 넘긴 경우에만 별도의 성 획득 세리머니를 이어서 표시한다.
- **별점 UI**: 0.5~5점, **0.5 단위(반별점)** (#153). 별 전체 탭 = 정수점, 별 좌측 절반 탭 = 0.5점 (SSOT: [profile.md §5.8.3](./profile.md))
- 미기록: 스트릭만 위기(방패 없으면 리셋). **둥지·XP·성은 존속**(v7 — XP 차감 폐지, [systems.md §6.2/§6.3](./systems.md))

### 5.5 읽기 모드 (#184) — 몰입 독서 + 상시 한 문장 캡처

둥지는 *진입*에 좋지만 읽으며 한 문장 적기엔 부적합 → 별도 **읽기 모드**(북모리식 몰입 화면).

- 진입 (v8 갱신, #432): 홈 CTA **"📖 읽기 시작"** → **같은 화면(홈)에 인라인으로 펼침**. 별도 진입 화면(2depth)·풀스크린 전환 없음 — `inline` prop으로 ReadingMode를 홈 책 카드 아래 렌더
- **독서 타이머**: 상단 경과 시간(⏱). 일시정지/계속. 백그라운드 시 Page Visibility로 일시정지(근사 정확)
- **상시 한 문장 입력칸**: 화면에 항상 열려 있어 떠오른 문장을 즉시 저장(아카이브). 저장마다 `sessions.addToday`(당일 1회 idempotent — 체크인/스트릭 충족) + `sentences.add`
- **저장 = 토스트만, 참새는 체크인(읽기 종료) 직후에만 (#347/#438)**: 읽는 중 저장은 토스트(흐름 유지) + `[저장]` 버튼만 — 인라인 '💬 지금 대화' 제거. 참새 질문은 **읽기 종료(✕/독서 종료/완독)** 시 세션 문장 1개로 1회, 말풍선 채팅 UI(#435) — 오늘 요약과 함께. SSOT: [companion-reading-end.md](./companion-reading-end.md)
- **인용 ↔ 내 생각 토글 (#360)**: 입력칸 위 「📖 책 속 문장 | 💭 내 생각」 칩. 의견도 한 문장처럼 가볍게 저장 — `sentences.kind`('quote' 기본 | 'thought'). 표시는 인용="이탤릭", 의견=💭. LLM엔 kind 전달(companion.md §2)
- **문장별 페이지**: 매 문장마다 "📍 이 문장의 페이지" 입력. **미입력 시 페이지 없이 문장만** 저장
- **읽기 모드가 오늘의 한 문장을 대체**: 읽기 모드에서 기록하면 둥지의 별도 "오늘의 한 문장"은 불필요(동일 체크인 경로)
- **종료**: "독서 종료" → 마지막 읽은 페이지 확인 입력 → `current_page` 갱신. 미입력 시 최근 페이지 유지
- 데이터: `DataStore.sessions.addToday`·`sentences.add`·`activeBook.get`. 누적 독서시간 저장은 Phase 2(`reading_sessions.seconds`)

### 5.6 책 사진 OCR 추출 (#382, 구현) — Stack Lock 해제(v8, 2026-06-11)

타이핑 마찰을 줄여 "한 문장 남기기"를 더 쉽게. **웹 기반 OCR**(네이티브 불필요 → Stack Lock 보류 대상 아님, [CLAUDE.md Stack Lock](../../CLAUDE.md)).

- **진입**: 읽기모드 인용(📖 책 속 문장) 모드의 입력칸 아래 **"📷 책 사진에서 글귀 가져오기"**. `<input type=file accept=image/* capture=environment>` → 모바일은 카메라/갤러리.
- **파이프라인**: 사진 → `POST /api/ocr`(Cloudflare Worker) → **① Upstage Document OCR**(`document-digitization`, `model=ocr`, 저렴·한글 우수) → **② solar-pro3 보정**(`temperature 0.1`): 책 좌우 단·줄바꿈으로 생긴 **불필요한 줄바꿈·띄어쓰기만** 이어붙임. **원문 단어 변형·요약·번역·맞춤법 교정 금지**(인용 충실도). 보정 실패 시 raw OCR 폴백(무중단).
- **드래그 영역 선택 v2 (#396, 구현)**: 사진 선택 → **크롭 오버레이**(`OcrCropOverlay`)에서 원하는 구절을 사각형으로 드래그 → **그 영역만 클라이언트 캔버스로 잘라** OCR. 배경 잡음·두 페이지 인터리빙(#395 벤치마크 발견)을 근본 회피. '전체' 버튼으로 통째 OCR도 가능. 포인터 이벤트(터치/마우스 공용), 좌표는 표시폭→naturalWidth 스케일 매핑.
- **촬영 가이드 (#395)**: OCR 버튼 아래 "한 페이지·한 구절만, 배경 없이" 안내 — 벤치마크 결론(Upstage OCR 본문 정확도 0.93, 변수는 *구도*).
- **결과**: 추출(보정) 텍스트를 입력칸에 **프리필(이어붙임)** → 사용자가 다듬어 저장. 여러 장 누적 가능.
- **키 보호**: `UPSTAGE_API_KEY`(companion과 동일 secret) **서버에서만**. 클라 노출 금지. 동일출처(Origin)만 허용. 이미지 ≤8MB.
- **내 생각(thought) 모드엔 미표시**: 내 생각은 직접 적는 것이라 OCR 불필요.
- 분석 이벤트: `ocr_extracted`(book_id, chars).
