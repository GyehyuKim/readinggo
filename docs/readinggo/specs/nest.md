### 5.1 둥지 탭 — 화면 구성

둥지 탭은 **활성 책 1권의 진행을 "둥지가 자라는" 모습으로** 보여준다. (구 The Path 경로 시각화는 v7에서 폐기.)

| 요소 | 동작 |
|---|---|
| 상단 바 | 🐦 로고 / 🔥 스트릭 / ⚡ XP / Lv / **🏰 ×N**(성 개수) / 📚(내서재 이동) |
| 🔥 스트릭 탭 | 탭 → 이달 달력 모달. 기록한 날짜에 🔥, 오늘 강조 |
| 🏰 배지 탭 | 탭 → 프로필 성 컬렉션([§5.8](./profile.md))으로 이동 |
| **책 진척 배너** | 활성 책 진척률 % + 미니 진척 바 + 책 표지·제목·현재/전체 페이지 (책 단위 — 둥지와 별개, #313) |
| **둥지 비주얼(메인)** | **누적 활동(XP)** 단계에 따라 **둥지가 점진적으로 자라는 일러스트**(🪵→🏰, §5.2). 책 무관·유지. 참새가 둥지에 머문다 |
| 활성 책 전환 | 진화 배너의 책 표지/제목 탭 → 활성 책 전환 시트(§5.3) |
| CTA (미완료) | "오늘 기록하기" → 미션 모달 (현재 페이지 입력 → 한 문장 입력) |
| CTA (완료 후) | "✍️ 한 문장 추가" — 추가 한 문장 입력 가능 (페이지 + 문장만, XP·스트릭 변동 없음) |
| 내 한 문장 | 활성 책에 내가 남긴 한 문장 목록(최신순). 캘린더 ↔ 리스트 토글 |

**하루 여러 문장**:
- 하루 첫 기록: 세션 생성 + 한 문장 저장 + XP +20 + 스트릭 +1 ([systems.md §6.1/§6.3](./systems.md))
- 이후 추가: 세션 재생성 없음. `sentences`에만 추가. XP·스트릭 변동 없음

**날짜 시뮬레이터 (Phase 0 데모 / 발표 전용)**:
- 우하단 플로팅 버튼 `🗓 YYYY-MM-DD +1일`
- 탭 시 `simDate` +1일 전진. 콕찌르기 제한 리셋
- 운영 빌드에서는 제거

### 5.2 둥지 진화 (Dynamic Stage) — 5단계

> **v7.4 변경 (2026-06-09, #313)**: 둥지를 **책에서 분리.** 기존 "활성 책 진척률" 모델 폐기.
> 둥지는 **누적 활동**에 따라 자란다 — 읽기·한 문장 입력·타인 한 문장에 짹(좋아요)→XP 등 **모든 행위가 반영**.
> **책과 무관하며 책을 바꿔도 유지**(1 유저 = 1 둥지). 둥지는 *전체 활동(고양 누적)의 시각화*다.
> 근거: 1책=1둥지면 둥지는 '한 권의 읽기 진척도'와 동일 개념 — 시각화·귀여움 외 의미가 없음.

성장 단계는 **누적 XP(레벨)** 구간으로 매핑. 단계 이모지·카피는 유지, 임계값은 [systems.md §6.3](./systems.md) XP/레벨 곡선 기준(구체 구간은 승원 합의 — TBD).

| 단계 | 이모지 | 단계명 | 색상 | 배경색 |
|---|---|---|---|---|
| 1 (시작) | 🪵 | 나뭇가지 자리 | #AFAFAF | #f3f4f6 |
| 2 | 🪹 | 빈 둥지 | #F59E0B | #FEF3C7 |
| 3 | 🏠 | 따뜻한 둥지 | #58CC02 | #F0FDF4 |
| 4 | 🏡 | 다정한 집 | #1CB0F6 | #EFF6FF |
| 5 | 🏰 | 참새의 성 | #CE82FF | #FAF5FF |

진화 시 마이크로카피:

| 전환 | 카피 |
|---|---|
| LV1→2 | "참새가 자리를 잡았어요!" |
| LV2→3 | "참새가 살림을 차렸어요!" |
| LV3→4 | "다정한 이웃이 되었어요!" |
| LV4→5 | "전설의 참새 성주!" |

- 누적 활동(XP)이 쌓이면 단계 상승. **책을 바꿔도 둥지는 유지**(활동 누적은 책 무관, #313). 완독한 책은 성(🏰) 컬렉션에 영구 보존(§5.2.1) — 둥지(활동 누적)와 성 컬렉션(책별 완독)은 별개 축.

#### 5.2.1 성(🏰) 컬렉션 — 영속 성취 지표 (v7 신설)

둥지 진화가 "이 책 진도"라 책마다 리셋되는 대신, **완성한 성을 모은다.**

- **완독(진척률 100%) = 🏰 1개 획득.** 완독 세리머니(§5.4) 후 성 컬렉션에 추가.
- **성 개수 = 누적 완독 권수.** 별도 카운터 없이 `user_books(status='completed')`에서 파생 ([backend.md §7.2/§7.3](./backend.md)).
- **전시 2곳**: ① 둥지 탭 상단 바 `🏰 ×N` 미니 배지(매일 노출), ② 프로필 성 선반(풀 컬렉션, [§5.8](./profile.md)). *구현은 nest가 원천 정의, profile이 따른다.*
- 각 성 카드 = 완독 책 표지 + 별점 + 완독일([§5.8](./profile.md) 완독 별점·소감).

### 5.3 활성 책 전환

여러 책을 동시에 읽을 수 있으나, **항상 한 권이 "활성 책"**.

- 활성 책 = 둥지 탭이 그리는 책
- 진화 배너의 책 표지/제목 탭 → **활성 책 전환 시트** 슬라이드업
- 시트 내용: 내 책장의 읽는 중 책 목록, 각 카드에 표지·제목·진척
- 카드 탭 → 활성 책 변경, 시트 닫힘, 둥지 비주얼이 해당 책 진척률로 재구성
- **각 책의 세션·진척·한 문장은 독립 보존**(책 전환 시 데이터 초기화 금지)
- 데이터: `users.active_user_book_id` ([backend.md §7.8](./backend.md)). 전환 = `DataStore.activeBook.set`
- **좌우 리볼빙 캐러셀 (#185)**: 둥지 책 카드에 `‹ ›` 화살표 + 하단 점 인디케이터. 읽는 중 책이 2권 이상이면 노출, 좌우로 넘겨 **즉시 활성 책 전환**(시트 안 거치고). 전환 = `window.RG_activateBook` → `activeBook.set` + 둥지 재계산

### 5.4 일일 미션 — [§4](./onboarding.md) D 참조

- 하루 1세션. 같은 날 재진입 시 "오늘은 이미 완료했어요" + 한 문장 추가 옵션(서브 액션, 스트릭·XP 영향 없음)
- 입력: 현재 페이지 + 한 문장(페이지 강제·문장 선택, **1000자 이내**). **그 문장이 속한 페이지로 기록**(진행률과 별개 개념 — "📍 N쪽의 문장")
- 완료 시 즉시 보상: XP +20 + 스트릭 +1 + 둥지 진척 갱신 ([systems.md](./systems.md))
- **완독 시(100%)**: 완독 세리머니 모달(🏰 + 완독 배지 + Confetti) → 별점·소감 입력([§5.8](./profile.md)) → 성 컬렉션 추가(§5.2.1)
- **별점 UI**: 0.5~5점, **0.5 단위(반별점)** (#153). 별 전체 탭 = 정수점, 별 좌측 절반 탭 = 0.5점 (SSOT: [profile.md §5.8.3](./profile.md))
- 미기록: 스트릭만 위기(방패 없으면 리셋). **둥지·XP·성은 존속**(v7 — XP 차감 폐지, [systems.md §6.2/§6.3](./systems.md))

### 5.5 읽기 모드 (#184) — 몰입 독서 + 상시 한 문장 캡처

둥지는 *진입*에 좋지만 읽으며 한 문장 적기엔 부적합 → 별도 **읽기 모드**(북모리식 몰입 화면).

- 진입: 둥지 CTA **"📖 읽기 모드"** → 활성 책으로 풀스크린 진입
- **독서 타이머**: 상단 경과 시간(⏱). 일시정지/계속. 백그라운드 시 Page Visibility로 일시정지(근사 정확)
- **상시 한 문장 입력칸**: 화면에 항상 열려 있어 떠오른 문장을 즉시 저장(아카이브). 저장마다 `sessions.addToday`(당일 1회 idempotent — 체크인/스트릭 충족) + `sentences.add`
- **문장별 페이지**: 매 문장마다 "📍 이 문장의 페이지" 입력. **미입력 시 페이지 없이 문장만** 저장
- **읽기 모드가 오늘의 한 문장을 대체**: 읽기 모드에서 기록하면 둥지의 별도 "오늘의 한 문장"은 불필요(동일 체크인 경로)
- **종료**: "독서 종료" → 마지막 읽은 페이지 확인 입력 → `current_page` 갱신. 미입력 시 최근 페이지 유지
- 데이터: `DataStore.sessions.addToday`·`sentences.add`·`activeBook.get`. 누적 독서시간 저장은 Phase 2(`reading_sessions.seconds`)
