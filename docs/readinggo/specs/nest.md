### 5.2 둥지 진화 (Dynamic Stage) — 1,600 XP 주기

> **v8.1 변경 (2026-06-15, #520)**: 전체 누적 XP를 그대로 단계에 매핑하는 모델을 대체한다.
> 둥지는 책과 분리된 XP 기반 시각화라는 원칙은 유지하되, **1,600 XP마다 성을 완성하고 새 둥지 주기를 시작**한다.

XP 값은 차감 없이 계속 누적한다([systems.md §6.3](./systems.md)). 둥지 표시만 현재 주기 값으로 계산한다.

```text
cycleXp     = totalXp % 1600
castleCount = floor(totalXp / 1600)
```

| 단계 | 현재 주기 XP | 이모지 | 단계명 | 색상 | 배경색 |
|---|---:|---|---|---|---|
| 1 (시작) | 0–99 | 🌿 | 나뭇가지 자리 | #AFAFAF | #f3f4f6 |
| 2 | 100–399 | 🪹 | 빈 둥지 | #F59E0B | #FEF3C7 |
| 3 | 400–899 | 🪺 | 따뜻한 둥지 | #58CC02 | #F0FDF4 |
| 4 | 900–1,599 | 🐣 | 다정한 집 | #1CB0F6 | #EFF6FF |
| 5 (경계 연출) | 1,600 도달 순간 | 🏰 | 참새의 성 | #CE82FF | #FAF5FF |

> 이모지는 토스트·세리머니 문구 등 텍스트 표면용. 화면 단계 아이콘은 `nestArt`(커스텀 SVG, #719). 둥지·새 테마로 통일(#756) — lv4 SVG는 집 → 부화 새끼.

- Lv5는 **1,600 XP 달성 시 도달하는 주기 완료 단계**다. 성 획득 후 현재 주기 XP는 `0`이 되고 둥지는 Lv1부터 다시 시작한다.
- 책을 바꾸거나 완독해도 `totalXp`와 현재 주기는 유지된다. 완독 자체는 성 획득 트리거가 아니다.

**진척 표시 = "절대 주기 XP / 다음 단계 임계값" 기준 (#743 — #682 갱신).** 진척 바·수치는 전체 주기(`cycleXp / 1600`)도, 단계-상대(`intoXp / spanXp`)도 아니라 **현재 주기 누적 XP를 다음 단계 임계값에 견준 절대값**으로 계산한다. 숫자와 바가 같은 기준이라 시각적으로 일치한다.

```text
stage   = getNestStageByXp(totalXp)         # 현재 단계
next    = 다음 단계 (없으면 최고 단계)
표시 숫자 = cycleXp / next.minXp            # 절대 누적 XP / 다음 단계 진입 임계값
bar pct  = cycleXp / next.minXp * 100       # 숫자와 동일 기준
```

- 예) 빈 둥지(100–399) 구간에서 `cycleXp=342` → `342 / 400 XP` (86%). 따뜻한 둥지(`cycleXp=471`) → `471 / 900 XP` (52%).
- **왜 #682(단계-상대)를 갱신했나**: 단계-상대는 `471`인데 화면에 `71 / 500`으로 나와 상단 XP(471)와 어긋났다(#743). 절대 기준은 상단 XP와 일치하고 숫자·바가 한 기준이라 직관적이다.
- **최고 단계 예외**: 다음 임계값이 없으면(`next=null`, `isMax`) **"최고 단계"**로 표기(`pct=100`). 실서비스에서는 1,600 경계가 곧 Lv1로 리셋되므로 방어적 처리.
- 공유 헬퍼 `nestStageProgress(totalXp)` → `{ stage, next, intoXp, spanXp, pct, isMax }` (data.js)는 그대로 두되, **표시 기준은 `cycleXp / next.minXp`** 다(nest.js `_absPct`/`_cycleXp`가 국소 계산). `intoXp/spanXp/pct`는 잔여 XP 문구 등 보조에만 쓴다. NestTheatre·체크인 세리머니 진척 바가 공통으로 절대 기준 사용.

진화 시 마이크로카피:

| 전환 | 카피 |
|---|---|
| LV1→2 | "참새가 자리를 잡았어요!" |
| LV2→3 | "참새가 살림을 차렸어요!" |
| LV3→4 | "다정한 이웃이 되었어요!" |
| LV4→5(주기 완료) | "전설의 참새 성주!" |

- XP가 쌓이면 현재 주기 안에서 단계가 상승하고, 1,600 XP 경계를 넘으면 성(🏰)을 획득한다(§5.2.1).

#### 5.2.1 성(🏰) — XP 주기 완료 지표

- **성 개수 = `floor(totalXp / 1600)`.** 별도 DB 컬럼이나 테이블 없이 `users.xp`에서 파생한다([backend.md §7.2/§7.3](./backend.md)).
- **획득 시점 = XP 추가 후 1,600 경계 통과.** Lv5 도달 세리머니 후 다음 주기를 시작한다.
- 완독 책은 [profile.md §5.8](./profile.md)의 읽은 책 목록에 그대로 남지만, 더 이상 성 개수나 성 카드로 환산하지 않는다.
- XP 차감이 없으므로 획득한 성은 줄어들지 않는다.

### 5.3 활성 책 전환

여러 책을 동시에 읽을 수 있으나, **항상 한 권이 "활성 책"**.

- 활성 책 = 홈 탭이 그리는 책
- **현재 구현 = 좌우 리볼빙 캐러셀 (#185, v7.2)**: 홈 책 카드에 `‹ ›` 화살표 + 하단 점 인디케이터. 읽는 중 책이 2권 이상이면 노출, 좌우로 넘겨 **즉시 활성 책 전환**. 전환 = `window.RG_activateBook` → `activeBook.set` + 둥지 재계산
- **좌우 스와이프 제스처 (#1001)**: 화살표 버튼과 **동일 전환 로직(`switchBook(dir)`)을 재사용**하는 터치·드래그 제스처를 책 카드에 추가(버튼은 접근성용으로 유지 — 제스처는 *대체가 아니라 추가*).
  - **방향**: **왼쪽으로 밀기 → 다음 책(`dir=+1`)**, **오른쪽으로 밀기 → 이전 책(`dir=-1`)**. 화살표(`‹`=−1·`›`=+1)·"내용이 손가락을 따라온다" 규칙과 일치.
  - **입력 통합**: Pointer Events(`onPointerDown/Move/Up/Cancel`) 하나로 모바일 터치 + 데스크탑 마우스 드래그 모두 처리. 마우스는 좌클릭만.
  - **임계치 ~45px**(`SWIPE_PX`) 넘으면 전환, 미만이면 **원위치 스냅백**. 8px(`SWIPE_SLOP`) 미만 이동은 탭으로 간주(오발동 방지).
  - **가로 우세일 때만 발동**(`|dx|>|dy|`): 첫 의미 이동에서 축을 잠가, **세로 우세면 제스처를 포기하고 페이지 세로 스크롤을 통과**시킨다. wrap `touch-action: pan-y`로 브라우저 세로 스크롤과 비충돌 보장. 가로 확정 후에만 `preventDefault`.
  - **드래그 중 모션**: 카드만 `translateX`로 손가락을 따라가되 0.6 저항 계수로 절제. 전환 확정/취소 시 `transform .2s ease-in-out`(DESIGN.md motion: short·move=ease-in-out, `transform`만 — 레이아웃 속성 금지)으로 스냅. 드래그가 실제로 일어났으면 카드 탭(책 상세 열기)을 1회 억제.
  - **책 1권 이하면 비활성**(스와이프 no-op, `touch-action: auto`로 복귀). 끝 책에서 더 밀면 `switchBook`의 모듈러 순환을 따른다(화살표 버튼과 동일).
- **각 책의 세션·진척·한 문장은 독립 보존**(책 전환 시 데이터 초기화 금지)
- 데이터: `users.active_user_book_id` ([backend.md §7.8](./backend.md)). 전환 = `DataStore.activeBook.set`
- **활성 책 중단 시 active 승계 (#643, #593 후속)**: 활성 책을 중단(`status='aborted'`)하면 active를 `null`로 해제만 하지 않고, 남아 있는 `reading` 책으로 **자동 승계**한다(승계 대상 = 캐러셀 첫 책 = `myBooks.list` 순서 첫 reading). 남은 reading 책이 없을 때만 `null` 유지(빈 상태가 올바름). 미승계 시 홈이 "책 등록하기" 빈 상태로 떨어지던 회귀 차단. 양 어댑터(localStorage `abort()` / Supabase `abort()`) 동일 보장.
- ~~구 설계: 표지/제목 탭 → 전환 **시트** 슬라이드업~~ → **캐러셀로 대체(시트 폐기, v7.2)**. 별도 `ActiveBookSheet` 컴포넌트 없음.

### 5.4 일일 미션 — [§4](./onboarding.md) D 참조

- 하루 1세션. 같은 날 재진입 시 "오늘은 이미 완료했어요" + 한 문장 추가 옵션(서브 액션, 스트릭·XP 영향 없음)
- 입력: 현재 페이지 + 한 문장(페이지 강제·문장 선택, **1000자 이내**). **그 문장이 속한 페이지로 기록**(진행률과 별개 개념 — "📍 N쪽의 문장")
  - **페이지 직접입력 편집 동작 (#686)**: 체크인 바텀시트의 페이지 직접입력 칸은 **빈 문자열·편집 중 부분값을 허용**한다. 매 키 입력마다 `0`으로 강제하거나 클램프하지 않아 백스페이스 삭제·커서 위치 편집이 정상 동작한다. 범위 클램프(`0 ~ book.total`)는 **블러/제출 시점**에만 확정한다.
- 완료 시 즉시 보상: XP +20 + 스트릭 +1 + 둥지 진척 갱신 ([systems.md](./systems.md))
- **체크인 세리머니 (v8 갱신, #449/#426)**: 스트릭 카드·"N일 연속" 문구 **제거** — 대신 **둥지 진척도 bar**(체크인 전→후 애니메이션)로 "오늘 한 쪽이 둥지를 키운다"를 보여줌. 둥지 단계가 오르면(`nestUp`) 별도 step의 **진화 축하 화면**(이전 단계 → 다음 단계 + 카피 + "계속 →")으로 전환
  - **진척 바 기준 = 현재 단계 구간 (#685, §5.2 헬퍼 재사용)**: 세리머니 진척 바도 `nestStageProgress`로 계산해 NestTheatre 표시와 일치시킨다. 하단 문구는 "성까지 N%" 대신 **다음 단계까지 남은 XP**(`{다음 단계명}까지 N XP`)로 안내한다.
  - **헤드라인 문구 분기 (#685)**: **페이지만 저장**(문장 미입력) 시 헤드라인은 **"책읽기 완료 +20 XP"**. **한 문장을 실제로 등록**한 경우에만 **"1개 문장 등록 +N XP"**. 페이지만 저장했는데 "문장 등록"으로 표기되던 거짓 문구 제거.
- **한 문장 보상 카드 정직 표시 (#549)**: 세리머니 보상 그리드의 한 문장 카드는 **문장을 실제로 남겼을 때만 `🔖 저장됨`**. 페이지만 업데이트(문장 미입력)했는데 "저장됨"이 뜨던 거짓 표시 제거 — 미입력 시 **이 책에서 모은 한 문장 수(`📖 N개 · 이 책 한 문장`)**, 0개면 **`✍️ 한 문장 남겨봐요` 독려**로 표시
- **완독 시(100%)**: 완독 세리머니 + 완독 배지 + Confetti(주기 완료 세리머니와 별개) → 별점·소감 입력([§5.8](./profile.md)). 완독 보상 XP가 1,600 경계를 넘긴 경우에만 별도의 성 획득 세리머니를 이어서 표시한다.
- **별점 UI**: 0.5~5점, **0.5 단위(반별점)** (#153). 별 전체 탭 = 정수점, 별 좌측 절반 탭 = 0.5점 (SSOT: [profile.md §5.8.3](./profile.md))
- 미기록: 스트릭만 위기(방패 없으면 리셋). **둥지·XP·성은 존속**(v7 — XP 차감 폐지, [systems.md §6.2/§6.3](./systems.md))

#### 5.4.1 마일스톤 회고 (v9, #938) — 서사로 고양감 증폭

게이미피케이션의 코어 감정은 '고양감(uplift)'이다. 점수·랭킹 같은 외적 보상은 과잉정당화로 내적 동기를 밀어내므로 **더하지 않고**, 이미 쌓인 **한 문장·순간**을 마일스톤에서 돌아보게 해 **숫자가 아니라 서사로** 고양감을 키운다. 완독 회고(참새, [companion.md](./companion.md) §4.1)와 별개 — 이쪽은 *전 기록을 가로지르는 내 흔적 타임라인*이다.

### 5.6 책 사진 OCR 추출 (#382, 구현) — Stack Lock 해제(v8, 2026-06-11)

- **드래그 영역 선택 v2.1 (#396·#791, 구현)**: 사진 선택 → **크롭 오버레이**(`OcrCropOverlay`)에서 원하는 구절을 사각형으로 드래그 → **그 영역만 클라이언트 캔버스로 잘라** OCR. 배경 잡음·두 페이지 인터리빙(#395 벤치마크 발견)을 근본 회피. '전체' 버튼으로 통째 OCR도 가능. 포인터 이벤트(터치/마우스 공용), 좌표는 표시폭→naturalWidth 스케일 매핑. **v2.1(#791): 그린 박스를 다시 그리지 않고 조정 — 안쪽 드래그=이동, 4모서리 핸들 드래그=리사이즈**(hit-test는 onDown 좌표계산, 핸들 시각 18px·히트 26px). 선택 정밀도 개선(줌 없이). - **결과**: 추출(보정) 텍스트를 입력칸에 **프리필(이어붙임)** → 사용자가 다듬어 저장. 여러 장 누적 가능.

### 5.7 배치 OCR — 사진에서 밑줄 모아 담기 (#844, 계획)

§5.6은 한 장씩 **글자만** 추출한다. **종이책에 밑줄/형광펜으로 표시하는 아날로그 독자**(본진)는 표시한 페이지가 여러 장 쌓이므로, 앨범에서 여러 장을 불러와 **각 장의 강조(밑줄/형광펜) 문장만 자동 추출** → 여러 줄로 정리 → 검토 → 일괄 저장한다. **뒷단(검토 큐 + 저장)은 §5.8 텍스트 일괄 입력과 동일 경로를 공유** — 사진이든 붙여넣기든 "문장 배열 → 검토 → 저장"은 같다.

- **진입**: 책 상세(BookDetailModal) **"📷 사진에서 여러 문장 담기"** — §5.8 텍스트 카드 옆(배치류 허브). 단발 OCR(§5.6)은 홈 입력부에 유지.
- **추출 = Gemini Flash vision (핵심)**: Upstage Document OCR은 글자만 읽고 "밑줄 위치"를 모른다(Solar는 텍스트 전용·vision 미지원, 2026-06 확인). 따라서 각 장 사진을 **Gemini Flash vision 서버리스 프록시**(`/api/extract-highlights`)에 보내 **"밑줄·형광펜 등 강조 표시된 문장만 추출"**(JSON 배열). **원문 충실**(병합·교정·요약·번역·환각 금지, §5.6 보정 원칙 재사용). 형광펜·연필·볼펜 모두 의미적으로 인식. Stack Lock: Gemini vision은 [CLAUDE.md AI v9](../../CLAUDE.md)로 자유(별도 결정 불필요).
- **형광펜 실동작 검증(#1006, 2026-06-26)**: 노랑·분홍·초록 형광펜 + 본문 혼재 합성 페이지(한글)를 배포 워커에 실호출 → 강조 3문장 **정확·원문 그대로 추출, 비강조 본문 3문장 전부 제외**. 색·다중 형광펜 처리 OK(밑줄과 동등). 프롬프트(`HIGHLIGHT_SYSTEM`)가 "강조된 것만"을 명시하고 비강조/머리말/쪽번호 제외를 지시 — 정확성 검증됨.
- **흐름**:
  ```
  [앨범 N장 선택]
     → 각 장: Gemini vision "강조 문장만 추출" (무료 10 RPM → 순차/소량 동시 + 지연, 진행률 표시)
     → 추출된 밑줄 문장들을 여러 줄로 정리
     → BatchQuoteImport 검토 큐(§5.8 공유): 편집/삭제/스킵
     → saveBatchQuotes: myQuotes 일괄 저장 + XP 합산 1회
  ```
- **공통 모듈(#848 공유)**: 검토 큐(`BatchQuoteImport`) + 저장(`saveBatchQuotes`)은 텍스트 import와 동일. OCR 배치 신규 코드는 **"앨범 다중 → 각 장 vision 추출 → 문장 배열"** 앞단뿐.
- **rate limit**: 무료 `gemini-2.5-flash` 10 RPM / 1,500 RPD. 수십 장은 순차+지연(또는 소량 동시)으로 조절. 실패 장은 스킵/재시도.
- **지역 precondition flap + 재시도(#1006)**: Gemini 무료 티어(`generativelanguage.googleapis.com`)는 송출 지역에 따라 `400 FAILED_PRECONDITION: "User location is not supported"`을 **간헐** 반환한다(워커가 도는 Cloudflare PoP의 egress 지역에 좌우 — 2026-06-26 실측 동일 사진 5회 중 ~50% 502 flap). 같은 요청이 다른 경로/PoP로 재송출되면 통과하므로 `callVision`은 **최대 3회 재시도**(429·5xx·`FAILED_PRECONDITION` 한정, 350ms×n 백오프, 멱등). 401/403 등 비일시 오류는 즉시 중단.
- **실패 vs 빈 결과 구분(#1006)**: 클라(`runOcrBatch`)는 장별 **비2xx·네트워크 실패 수(`failed`)**를 집계해, 추출 0건일 때 서버/지역 outage(`failed≥N`)는 "추출에 실패했어요 — 잠시 후 다시"로, 강조가 실제로 없을 때만 "더 또렷한 사진으로"를 보여준다(backend 장애를 사진 탓으로 오인 표시 금지). 일부 장 실패+일부 성공이면 찾은 문장은 보여주되 실패 장 수를 안내.
- **환각 안전망**: vision이 없는 문장을 지어낼 위험 → 검토 큐에서 사용자가 확인·삭제로 거른다. (#1006 검증: 비강조 본문은 추출되지 않아 over-extraction 환각은 관측되지 않음.)
- **키 보호/제한**: `GEMINI_API_KEY` **서버에서만**(클라 노출 금지). `VISION_BASE_URL`/`VISION_MODEL`은 wrangler vars + 로컬 `.env`. 이미지 ≤8MB(§5.6 동일). 동일출처만.
- **분석 이벤트**: `ocr_batch_started`(count), `highlights_extracted`(page_idx, n), `ocr_batch_saved`(book_id, saved, skipped).

### 5.8 비정형 문장 텍스트 일괄 입력 (#848, 계획)

§5.6/§5.7은 **이미지 → 텍스트**다. 본 건은 **텍스트 → 문장**: 이미 텍스트로 가진 글귀 뭉치(전자책 메모, 타 앱 발췌, 독서 중 적어둔 메모 등)를 **붙여넣으면 개별 "한 문장"으로 분리해 일괄 등록**한다. 2026-06-20 수동 dry-run(헤일메리 43문장 콘솔 주입, 저장 43/실패 0)으로 저장 경로를 실증함 — #848 코멘트 참조.

- **진입**: 한 문장 입력 영역에 "📋 여러 문장 붙여넣기" → 텍스트 영역에 뭉치 붙여넣기.
- **분리 (2단)**:
  - **1차 단순 분리**: 줄바꿈/빈 줄 기준 split + trim + 빈 줄 제거(정형 입력은 이걸로 충분 — dry-run이 이 경로).
  - **2차 LLM 파싱(비정형)**: 문단 섞임·OCR 잡음·구분 없는 뭉치는 서버리스 프록시로 LLM에 문장 배열(JSON) 구조화 요청. **원문 충실 — 병합·교정·요약·번역·환각 금지**(OCR §5.6 보정 원칙 재사용). 짧은 대사·비문·오타도 보존. LLM은 "분리"만, 내용 생성 금지.
- **검토 큐**: 분리 결과를 리스트로 보여주고 **개별 편집/삭제/스킵** → 확정분만 저장(§5.7 리뷰 큐와 동일 UX).
- **제약/검증(dry-run 확인)**: `text` **1~200자 CHECK** — 200자 초과 조각은 경고·분할 유도. `page`는 미상이면 **null**. 기존 문장과 text 비교로 **중복 스킵(idempotent)**.
- **저장 — 공통 모듈(#844와 공유)**: "다건 → 검토 큐 → myQuotes 일괄 append + XP 합산 1회 + 완료 요약 1회"는 §5.7 OCR 배치와 **동일 백엔드 경로**. `handleCheckin` N회 호출 금지(세리머니·스트릭 폭주 회피). 공개범위(visibility)는 일괄 주입 전 선택(기본 public이라 명시 UI 필요 — 피드 노출/스포일러/저작권).
- **LLM 엔드포인트**: Gemini Flash 무료 티어 또는 solar-pro3 재사용, **서버리스 프록시**(예: `/api/parse-sentences`). 클라 키 노출 금지([CLAUDE.md Stack Lock](../../CLAUDE.md)).
- **분석 이벤트**: `text_import_opened`, `text_import_parsed`(method=split|llm, count), `text_import_saved`(book_id, saved, skipped).
