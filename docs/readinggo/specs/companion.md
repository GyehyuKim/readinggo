## 1. 포지셔닝 (What's New)

> **"혼자 읽어도, 한 문장을 남기면 '왜 그게 좋았어?'라고 물어봐 주는 — 언제나 열려 있는 1인 독서모임."**

- **What's New**: 독서의 기록 단위를 '몇 쪽·완독'이 아니라 **'한 문장이 나를 끌어올린 순간 + 왜'** 로 바꾼다. 남기면 끝(저장)이 아니라 **시작**(질문 → 대화).
- **What's Different (북모리 대비)**: 북모리는 문장을 *저장*한다. ReadingGo는 문장으로 *대화*한다. (북모리 AI 기능 없음 — 클린한 차별점)
- **Why You (해자)**: 콘텐츠가 아니라 **내 축적 맥락**(문장·왜·이력)에 기반한 관계. 쓸수록 나를 더 아는 진행자 = 인터랙션 해자.
- **코어 감정**: 고양감(uplift) — 질문이 그 순간을 한 번 더 끌어올린다.

### 1.1 "그냥 ChatGPT 아냐?" 방어 (피치 필수)

| | ChatGPT | ReadingGo 파트너 |
|---|---|---|
| 시작 | 내가 시켜야 함 | **먼저 물어봄** (내가 남긴 그 문장에 대해) |
| 지속 | 휘발 | **쌓임** → 갈수록 나를 더 아는 진행자 |
| 맥락 | 책 밖 | **읽는 흐름 안**, 내 문장이 재료 |

```
읽는 중: 한 문장 저장 → 토스트만 (마찰 제로, 흐름 유지)
      ↓
읽기 종료 (✕ 나가기 / 독서 종료 / 완독)
      ↓
참새: 세션에서 모은 문장 중 하나 선택 → "왜 이 문장이었어요?"   ← 1세션 1질문, 말풍선(좌)
      ↓
답한다 (말풍선 우측, 또는 건너뛰기 — 페널티 없음)
      ↓
답을 받아 한 걸음 더 깊이 되묻는 후속 질문 (멀티턴, 무료 5턴) ✅ #327/#655
      ↓
대화가 그 문장 my_note로 저장(Q/A) → 책 상세의 '내 한 문장'에 축적(#439)
```

- **문장 선택 (세션 내 여러 개)**: ① 코멘트(감상) 달린 문장 우선 → ② 없으면 마지막 저장 → ③ 0개면 참새 미등장. 읽기 종료 화면에 **오늘 요약(읽은 쪽·문장 수) + 참새 질문**이 한 화면.
- **멀티턴 대화 (✅ 구현 #327, 캡 갱신 #655)**: 질문 → 답 → 후속 질문 → … **무료 5턴**(이전 3턴). 매 답마다 대화를 해당 문장 my_note에 저장. 후속은 `/api/companion`에 `exchanges`(이전 Q/A) 전달.
  - **종료 = 이탈(#655)**: 진행 중 '마치기' 버튼을 **두지 않는다** — '대화 끝내기'를 별도 액션으로 두면 오해로 조기 종료하는 행동을 유발. 멈춤은 그냥 모달을 닫거나(✕) 바깥으로 나가는 것. 각 턴이 이미 my_note에 저장되므로 직전 턴까지 보존. 답을 타이핑만 하고 안 보낸 채 닫으면 그 미전송 답만 사라짐(허용).
  - **5턴 도달(#655)**: 끊기는 느낌 대신 따뜻한 마무리 한 줄(예: "오늘 재키랑 깊이 이야기했네요 🐦"). **수익화 시점에 이 지점이 '더 이야기하기 = 업그레이드' 훅**으로 전환. 무제한 티어 도입 시 워커 `exchanges.slice(0,6)` 상향 + 어뷰징 레이트리밋을 함께 검토(5턴까지는 slice 안에 들어와 현행 안전).
  - **턴 수 표시(#654)**: 홈 '내가 남긴 흔적' 카드의 재키 진입 버튼에 그 문장의 대화 턴 수 `(N)`을 노출(N = my_note의 `Q.` 블록 수, 책 상세 `library.js` 표기와 동일 계산·어휘). 0이면 숨김.
- **대화형·짧게·서식없음 (#406/#408)**: 후속 질문은 **사용자의 방금 답변에 먼저 반응**하고 그 답을 실마리로 한 가지만 되묻는다(답이 짧거나 농담이어도 무시 금지). 전체 2~3문장·질문 하나. **마크다운 서식 금지**(시스템 프롬프트 + 서버 `stripMd` 안전망 — UI가 md 미렌더라 별표 날것 노출 방지). 화면 안정(#407): 대화 말단 anchor를 view로 고정해 답변 생성 시 점프·오탭 방지.
- 질문은 **책 맥락 + 개인 맥락(왜 걸렸나)** 을 함께 건드린다. 책·작가 지식 곁들임.
- **인용 vs 감상 구분 (#359)**: LLM은 "한 문장"을 **작품 속 인용**으로, "메모"를 **사용자의 감상**으로 받는다(프롬프트에 역할 명시). 인용으로 보기 어렵거나(예: "즐거웠다") 작품 내 맥락을 모르면 **함부로 단정하지 않고 독자에게 맥락을 되묻는다**. 입력 단계 구분 **구현(#360)**: 읽기모드 토글로 `kind`(quote|thought) 저장 → `/api/companion`에 전달. thought면 "독자 본인의 생각"으로 명시되어 인용 규칙 대신 생각 자체를 여는 질문.
- **종류 자동 추정 (#420)**: 토글 기본값 `'quote'`로 인해 비교·감상 문장이 인용으로 오분류되던 문제 완화. 입력칸에 타이핑할 때마다 `estimateSentenceKind()`(`nest.js`)가 텍스트를 보고 토글을 **prefill**한다 — 따옴표로 시작/끝나면 `quote` 유지, "~같다/닮았다/생각한다/느꼈다" 등 메타·1인칭 서술이나 다른 작가·작품·책 언급이 보이면 `thought`로 제안. **오버라이드**: 사용자가 토글을 한 번이라도 직접 누르면 그 세션(해당 문장)에서는 자동 추정이 더 이상 덮어쓰지 않음. **OCR 경로(#382)**: 책 사진에서 추출한 텍스트는 원문 인용이므로 추정과 무관하게 `quote` 유지. 저장 후 다음 문장은 추정 상태가 초기화됨.
- **스킵/건너뛰기** 가능 — 강요하지 않음 (진입 마찰↓, 오피스아워 §4). 종료는 별도 '마치기' 액션이 아니라 이탈(✕/바깥)로 통일(#655). 미동의 시 로컬 목 단발(멀티턴 X).

## 4. LLM 연결 — Upstage Solar (구현 완료, #287)

- **provider-agnostic 클라이언트**: `worker/index.mjs`의 `callLLM()`이 `LLM_BASE_URL`·`LLM_MODEL`·`UPSTAGE_API_KEY`를 **전부 env에서** 읽음(하드코딩 금지). OpenAI 호환 `chat/completions`. → Gemini·Ollama 전환 시 env만 교체.
- **현재 모델**: `solar-pro3` (Upstage, `https://api.upstage.ai/v1`). 인터랙티브 질문 품질용. 대량 프로파일링은 더 싼 티어로(후속, `LLM_MODEL`만 교체).
- **reasoning 토글**: `LLM_REASONING_EFFORT` — 미설정/빈 값=추론 최소(기본). `low|medium|high` 설정 시 `reasoning_effort`로 전달. 질문이 밋밋하면 env만 올림(코드 수정 없음).
- **라우트**: `POST /api/companion` (`worker/index.mjs`). 입력 `{sentence, bookTitle}` → 출력 `{question}`. system = 독서 회고 진행자(한국어 질문 1개).
- **키 보호**: 키는 **서버(Worker secret)에서만**. `wrangler secret put UPSTAGE_API_KEY`. 클라 번들·React CDN 노출 금지. `LLM_BASE_URL`·`LLM_MODEL`은 `wrangler.toml [vars]`.
- **graceful fallback**: 키 없음/호출 실패 시 **목 질문으로 폴백**(서버·클라 양쪽). 데모·피치 무중단.
- 클라(`nest.js`): 저장 직후 `genCompanionQuestion()`이 `/api/companion` 호출(로딩→질문, 실패 시 목).

### 4.1 완독 회고 모드 (#259, 구현 완료)

### 4.4 질문 방향성 프리셋 (#375, 구현 완료)

사람마다 선호하는 질문의 난이도·관점이 다름. **자유 프롬프트가 아니라 정해진 프리셋(취향 칩)** 채택 — 허들·악용·일관성 리스크 회피(자유서술은 후속 고급옵션, Ref #287).

- **프리셋(7종)**: `balanced`(균형, 기본) · `deep`(깊이 파고들기) · `light`(가볍게) · `emotional`(감정 중심) · `critical`(비판적) · `context`(작가·맥락) · `author`(작가의 시선, #935). 정의 = `config.js` `RG_COMPANION_PRESETS`.
  - **`context` vs `author` (다름)**: `context`는 작가 *맥락*(삶·시대·작품 배경)과 **연결**, `author`는 "작가라면 이 장면을 왜 이렇게 썼을까" 식 작가 *시점* **추론**. 두 결을 별개로 둔다.
  - **`author` = #922 작가 시점의 사용자 선택화 (#935, §4.5)**: #934 가 `COMPANION_SYSTEM`에 넣었던 *자율(가끔 자동)* 작가 단락을 제거하고, 그 결(작가 시점 + 가드 3종)을 `PRESET_TONE['author']`로 옮겨 **`author` 선택 시에만** 작가 시점이 되게 한다(자율 → 선택).
- **저장**: 디바이스 선호(테마류) → `RG_companionPreset`(localStorage `rg_companion_preset`). 사용자 콘텐츠 아니므로 Supabase 미동기(로컬). 기본값 `balanced`.
- **UI**: ① 설정(`SettingsModal`) 칩 토글 + ② **재키 대화 화면(`CompanionModal`) 헤더 아래 칩 스트립(#935)** — 대화 맥락에서 '작가의 시선' 등으로 **즉시 전환**. 두 표면 모두 `RG_COMPANION_PRESETS` map(프리셋 추가 시 자동 노출)·동일 칩 패턴(선택=브랜드 솔리드/비선택=라인, DESIGN.md 위계 일관). 고른 결은 **다음 질문부터** 반영.
- **전달**: `genCompanionQuestion`·`genCompanionFollowup`이 호출 시 현재 프리셋을 읽어 `/api/companion` body `preset`으로 전송(호출부 시그니처 불변).
- **주입**: worker `PRESET_TONE[preset]`(키는 `RG_COMPANION_PRESETS`와 1:1)을 질문 지시문에 한 줄 덧붙임. `balanced`는 키 없음(기본 톤 유지). 시스템 프롬프트 본문은 불변.

### 4.5 책 지식 — "같이 읽은 진행자" (#656)

재키가 *그 책을 같이 읽은 독서모임 진행자*처럼 느껴지는 것이 핵심 UX다. 한 문장 집중은 유지하되(다른 문장·과거 대화는 주입 안 함 — 그 종합은 회고 §4.1·되감기 §5가 전담), **그 책/작가 지식**은 깊고 일관되게 깐다.

- **현재 한계(개선 전)**: `fetchBookBrief`(`worker/index.mjs`)가 책 소개(#1044 소스 이전: 카카오 키 설치 시 카카오 `contents`, 없으면 레거시 알라딘 `description` — 뒷표지 마케팅 카피)만 400자로 쓰고, **첫 질문에만**(`exchanges.length===0`) 붙임. 각 호출이 stateless라 **2턴부터 책 맥락을 잊고** 질문이 일반화됨.
- **격상안**:
  - **LLM 문학 브리프**: 알라딘 소개를 *그라운딩*으로 넣어 LLM이 주제·작가·시대·톤을 담은 짧은 브리프로 압축. 마케팅 카피 그 자체를 주입하지 않음.
  - **책별 캐시**: 같은 책의 후속 호출에서 브리프를 재생성·재조회하지 않음(비용·지연↓). 캐시 키 = 책(제목/ISBN).
  - **전 턴 주입**: 브리프를 첫 턴뿐 아니라 **모든 후속 턴 컨텍스트**에 포함 → 대화가 깊어져도 책을 잊지 않음.
  - **톤**: `COMPANION_SYSTEM`을 "같이 읽은 동료 진행자" 결로 미세 강화. 핵심은 여전히 그 사람의 경험·감정과 잇는 것.
  - **환각 방지**: 모르는 책·불확실한 장면은 단정하지 말고 독자에게 맥락을 되묻는다(#359 규칙 유지). 고전 위주 카탈로그라 리스크는 낮음.
  - **작가 시점 — 자율(#922/#934) → 사용자 선택(#935)**: 외부 평가자가 "작가와의 대화"를 주목 → 재키가 "작가라면 이 장면을 왜 이렇게 썼을까" 식 작가 시점을 곁들인다. **초기(#922/#934)**: `COMPANION_SYSTEM`에 *가끔 자동(자율)* 으로 넣음. **갱신(#935)**: 자율 단락을 제거하고 **질문 결 프리셋 `author`('작가의 시선')로 사용자 선택화**(§4.4) — 결·가드를 `PRESET_TONE['author']`로 이동, `author` 선택 시에만 작가 시점. 새 DB 추가 없음(프롬프트 한 줄 + 기존 프리셋 칩에 1종 추가 + 대화 화면 칩 전환). **가드(보존)**: ① 작가 1인칭 사칭 금지("나는 [작가]다" 금지) — "작가라면 이렇게 봤을 수도 있다"는 작품 근거 추론으로만, ② 작가가 실제로 안 한 말·생애 사실 단정 금지, 모르면 되묻기(#359와 동일). demand 미검증이라 프롬프트만으로 wedge 검증(통하면 그라운딩 강화는 후속).
- **graceful fallback**: 키 없음·조회/생성 실패 시 브리프 없이 진행(목 폴백 포함) — 데모 무중단.

### 4.6 문장별 "내 감상만" vs "재키와 대화" (#1070, 구현)

베타 피드백: *"재키 안 쓰고 내 감상만 쓸 수 있게끔. 한 문장별로 설정하기."* 남길 때마다 재키 대화로 끌려가지 않고, **그냥 내 감상만 적고 끝낼 수 있는 길**을 **문장 단위**로 연다. 재키는 권유지 강제가 아니다(§2 "스킵/건너뛰기 가능"의 명시화 — 코어는 *한 문장 성찰*이지 AI 가 아님).

- **두 모드 (문장별)**: `note`(내 감상만 — 재키 질문 없이 자유 메모) · `jacky`(재키와 대화 — 현행 멀티턴). `CompanionModal` 헤더 아래 **세그먼트 토글**로 한 탭 전환(선택=브랜드 솔리드/비선택=라인, DESIGN.md 위계). 감상 모드에선 재키 아바타·질문 결 칩·답변 입력바를 숨기고, 자유 메모 textarea + `내 감상 저장`(1차 솔리드)만 보인다. 헤더도 모드별로 바뀐다(감상=만년필 아이콘·"내 감상", 재키=재키 아바타).
- **진입 — 문장별 선택을 카드에서 분명히**:
  - 홈 '내가 남긴 흔적' 카드 — `내 감상`(2차 tonal) · `재키와 대화`(현행 q-ai) 두 버튼을 나란히. 감상이 있으면 카드에 미리보기 + 라벨 `감상 수정`. 재키 턴 수 `(N)` 표기는 유지(#654).
  - 책 상세 '내 한 문장' — 동일한 두 진입(미리보기는 자유 감상 블록만 보여줌).
  - `RG_openCompanion(sentence, { mode })` 로 모드를 명시해 연다. 모드 미지정 진입(되감기·모아보기 등)은 그 문장이 이미 가진 성찰(`rgNoteKind`)로 추정 → 없으면 아래 기본값.
- **데이터 — 한 `my_note` 칸, 블록 분리 (과거 #404 충돌 해소)**: 자유 감상 편집은 한때 폐지됐다 — 자유 감상과 재키 Q/A 가 같은 `my_note` 칸을 서로 덮어썼기 때문(#404). 이제 `my_note` 를 **블록으로 분리**한다: 자유 감상 블록(`Q.` 로 시작하지 않음) + 재키 Q/A 블록(`Q. …\nA. …`). `rgSplitNote`/`rgJoinNote`(`companion.js`, `window` 노출)가 저장 시 **상대 영역을 항상 보존** — 재키 턴 저장이 감상 블록을 보존하고, 감상 저장이 Q/A 블록을 보존한다. `parseNoteToExchanges` 는 이미 Q/A 블록만 추출하므로 재키 화면·턴 수·되감기 후보(Q/A 필요, §5)는 불변.
- **비강제·완결**: 감상만으로 성찰이 완결된다 — `DataStore.sentences.setNote(id, note)`(localStorage·Supabase **양 어댑터 표면 일치**, [backend.md §7.2](./backend.md)) 한 번으로 영속하고 `rg:sentence-note` 이벤트로 둥지·책 상세에 즉시 반영. LLM(재키) 호출 없음. 분석 이벤트 = `reflection_note_saved`.

**기본 모드 (#1081 결정 = `note`)**: 빈 문장을 **모드 미지정**으로 열 때 어디로 떨어질지(`RG_REFLECT_DEFAULT`, `companion.js`). 카드의 두 버튼은 어차피 모드를 명시하므로 이 기본값은 **카드 밖 진입**(되감기·모아보기 등)에만 영향한다.

| 후보 | 동작 | 메모 |
|---|---|---|
| **`note`** (채택, #1081) | 감상칸을 먼저, 재키는 한 탭 옆 | 베타 피드백 직접 수용 — 카드 밖 기본은 '내 감상' |
| `jacky` | 재키를 먼저 노출, '내 감상'은 한 탭 옆 | 대안(코어 '대화' 서사 보존). 카드 버튼으로 여전히 직접 진입 가능 |

**결정 (#1081, 계휴) = `note`**(`RG_REFLECT_DEFAULT='note'`) — 카드 밖 진입 기본은 '내 감상'.
