### 5.8 프로필(내서재) 탭

```
┌─────────────────────────────────────┐
│ @계휴  · 한 줄 소개       [⚙️ 설정]   │
│        ⚡ 1,240 XP   Lv.5            │   ← 헤더 스탯
├─────────────────────────────────────┤
│         🐦 둥지 캐릭터(NestTheatre)   │   ← v8: 홈에서 이동(#428)
├─────────────────────────────────────┤
│ 📖 현재 읽는 중  사피엔스 5/300p      │
│ [검색창]                             │
│ 읽는 중 │ 읽은 책 │ 중단 │ 관심 책   │   ← 서재 서브탭
│ [데미안] [82년생…] [불편한 편의점]    │
│ 📗 데미안 228/228p · 완독 ⭐4         │
└─────────────────────────────────────┘
```

- 상단: 닉네임·한 줄 소개(편집), 팔로워/팔로잉, **⚡ XP·Lv** 스탯 + 검색 (한 줄로 압축, 높이 절약). **v8**: 🏰성·🔥스트릭 배지는 헤더에서 제거(#425/#441)
  - **드리프트 정정 2026-07-09**: 위 ASCII·서술과 달리 코드(`library.js` 헤더)의 실제 스탯은 **팔로잉 / 팔로워 / 좋아요** 3개뿐 — **⚡ XP·Lv 스탯은 헤더에 없다**(둥지 캐릭터·상단바로 일원화). 헤더 **검색창(`[검색창]`)도 없다** — 책 찾기는 '내 서재' 타이틀 아래 하단 **'책 찾아 담기'** 버튼(#1060)으로 대체되었다.
- **(v7.2 QA5, #205)**: 상단과 중복되는 중간 스탯 카드(둥지레벨/완독/스트릭/XP) **제거**, 성 카드 별점 **유효숫자**(4→4.0)
- **둥지 캐릭터(NestTheatre, v8 #428)**: 헤더 바로 아래 렌더. 현재 1,600 XP 주기의 5단계([nest.md §5.2](./nest.md)) 일러스트 — LV·🔥 배지는 표시 안 함(헤더로 일원화), 전체 XP와 상단바 레벨은 리셋하지 않음
- **둥지 단계 이름 표시 + 가이드 (#511, 아이콘 #585)**: 둥지 라벨은 숫자 Lv 대신 **이모지 + 단계명**(예: `🪹 빈 둥지 → 🏠 따뜻한 둥지`)으로 표시. **가이드 버튼은 ghost circle `ⓘ`**(얇은 테두리 + 투명 배경, plain `?` 폐기 — 버튼으로 인지되도록) — 단계명 행(`nest-name-row`) **우측 끝**(진척 XP 표기 옆)에 고정해 "이 단계가 뭔지 궁금하면 탭" 맥락을 잇는다. 탭 → **단계 안내 팝업**(`NEST_STAGES` SSOT 기준 5단계 + XP 구간 + `🏰 1,600 XP 달성 → 성 획득, 나뭇가지 자리부터 새 사이클` 안내, **LV 표기 없이** 이모지·단계명·XP만). 현재 단계는 팝업에서 강조. **단계명 행 줄바꿈 (#621)**: 다음 단계(`→ 🏰 참새의 성`)와 진척 XP 표기는 **구(phrase) 단위로만 줄바꿈**(`white-space: nowrap`) — 좁은 폭에서 '성'·'XP'가 한 글자만 다음 줄로 떨어지는 고아(orphan)를 방지

- **한 문장 기본 공개 범위 (#1261, 개인정보·데이터)**:
  - 항목명은 `한 문장 기본 공개 범위`, 선택지는 **전체 공개**(`public`, "피드에 공개돼요")와 **나만 보기**(`private`, "나만 볼 수 있어요") 두 가지다. 초기 선택은 현행과 같은 **전체 공개**다. 문장별 공개 범위 SSOT와 친구 공개(`followers`)는 [feed.md §5.7.1](./feed.md#571-한-문장-노출--페이지-블라인드자동--공개-범위-3단계-v72-ssot)을 따른다. `followers`는 이 기본 설정의 선택지에 추가하지 않으며, 기존 문장별 순환 칩에서는 계속 선택할 수 있다.
  - **적용 시점·범위**: 설정 변경은 저장 성공 이후 생성되는 한 문장의 입력 초기값에만 적용한다. 홈 체크인, 책 상세 추가, 배치/OCR·타사 가져오기 등 `DataStore.sentences.add`를 쓰는 모든 신규 등록 경로는 폼을 열거나 초안을 만들 때 저장된 기본값을 읽고, 사용자가 제출 전에 문장별 공개 범위를 바꾸면 그 명시값이 우선한다. 설정을 다시 바꿔도 열려 있는 초안의 명시값과 이미 저장된 문장은 바꾸지 않는다.
  - **기존 데이터·마이그레이션**: 설정 키가 없거나 값이 잘못됐으면 `public`으로 해석한다. 기능 도입 시 기존 사용자에게 설정 키를 일괄 백필하지 않고, 기존 `sentences.visibility` 행도 갱신하지 않는다. DB 컬럼 기본값 `public`과 기존 공개 범위 마이그레이션은 그대로 유지한다.
  - **게스트·로그인**: 게스트는 `DataStore.settings`의 로컬 어댑터(`rg_v41`)에 저장해 같은 기기·브라우저 재시작 뒤 유지한다. 로그인 사용자는 같은 계약의 Supabase 어댑터로 `users.settings.default_sentence_visibility`에 저장해 계정 범위로 유지한다. 게스트 설정을 로그인 계정으로 자동 이관하거나 병합하는 것은 이 이슈 범위가 아니다. 로그인 전후 어댑터가 바뀌면 각 저장소에 존재하는 값(없으면 `public`)을 사용한다.
  - **UI·접근성**: 두 선택지는 라디오 그룹 또는 동등한 단일선택 컨트롤로 제공하고, 그룹명·현재 선택을 스크린리더가 읽을 수 있어야 한다. 키보드 방향키/Tab/Space로 선택 가능하고 포커스 윤곽을 표시하며, 텍스트 라벨만으로 의미가 구분되어야 한다. 색·아이콘만으로 공개 상태를 전달하지 않는다. 문장 등록 UI에도 현재 초기값을 텍스트로 표시하고 제출 전에 기존 문장별 공개 범위 컨트롤로 변경할 수 있게 한다.
  - **저장·오류**: 선택 즉시 `DataStore.settings.update`를 호출하되 저장 완료 전에는 확정 상태로 표시하지 않는다. 실패하면 이전 선택으로 되돌리고 `기본 공개 범위를 저장하지 못했어요. 다시 시도해 주세요.`를 인라인 또는 토스트로 알린다. 설정 읽기 실패는 안전한 호환 기본값 `public`을 사용하되 오류를 조용히 삼키지 않고 재시도 가능한 안내를 노출한다. 문장 저장 실패 시 기존 쓰기 실패 계약([backend.md §7.2](./backend.md#72-datastore-계약-phase-0--phase-1-이음매--v7-신설))을 따르며, 실패한 문장을 공개 또는 비공개로 저장된 것처럼 표시하지 않는다.
  - **수용 경계**: 새 계정·설정 키 없는 기존 계정·새 게스트는 `public`; `private` 저장 후 앱 재시작 시 선택 유지; 이후 신규 문장은 `private`; 제출 전 개별 `public`/`followers` 변경 시 그 값으로 저장; 설정을 `public`으로 되돌린 뒤 신규 문장은 `public`; 어느 경우에도 기존 문장과 열린 초안은 비변경이어야 한다.
  - **구현 파일 소유권**: 설정 UI는 `docs/readinggo/js/settings-modal.js`, 설정 조회·저장과 문장 생성 정규화는 `docs/readinggo/js/datastore.js`·`docs/readinggo/js/datastore-supabase.js`가 담당한다. 신규 문장 호출부는 추적된 `docs/readinggo/js/app.js`(홈·배치·가입 전 동기화), `docs/readinggo/js/book-detail-modal.js`(책 상세·배치), `docs/readinggo/js/data-import.js`(타사 가져오기)가 소유한다. 구현 PR은 이 spec-only PR 머지 후 별도로 연다.
  - **비목표**: 기존 문장 일괄 변경, 피드/RLS 공개 판정 변경, `followers` 기본 옵션 추가, 감상 `note_private` 기본값 변경, 게스트→계정 설정 이관, DB 마이그레이션·새 테이블/컬럼·의존성 추가는 하지 않는다.
- **스트릭 리마인더 (#1033, 읽기 환경)**: 매일 정해진 시각에 *로컬* 알림으로 "오늘의 한 줄"을 권한다 — `@capacitor/local-notifications`(공식 플러그인, 서버·FCM 불필요, 기기 내 스케줄). **네이티브 앱에서만 노출**(웹/데모는 의미 없어 토글 숨김 — `window.RG_streakReminder.isNativeSupported()` 가드). 켜기/끄기 + 시각 선택(기본 **21:00**, 기본 **OFF**). 켤 때 OS 알림 권한을 요청(첫 진입 강요 대신 토글 ON 시점), 거부 시 토글 원복. 설정은 `localStorage` `rg_streak_reminder`(`{enabled,hour,minute}`). **중복 방지**: 오늘 이미 기록했으면(`DataStore.streak.get().last_check_in_date===오늘`) 오늘치는 건너뛰고 내일 같은 시각으로 — 앱 부팅·resume마다 재무장(`RG_streakReminder.reschedule()`)해 상태를 최신화. 스케줄은 고정 알림 id 1건('다음 발생 시각' + `repeats:true`로 앱 미실행 시에도 매일 발화). 네이티브 변경이라 **새 APK 재빌드 필요**(OTA 불가).

- **탭바 설정 액션화 (#567)**: 하단 탭바의 `⚙️ 설정` 항목은 목적지 탭이 아니라 **바텀시트를 여는 액션**. `activeTab`에 `'settings'`를 넣지 않는다. 시트가 열린 동안 `settingsOpen` 상태로만 가벼운 pressed 스타일(`opacity: 0.7`) 부여 — 다른 탭의 `.active` 하이라이트와 구분. 시트 닫으면 직전 `activeTab`으로 복귀.

#### 5.8.1 성(🏰)과 읽은 책 분리

- **성 개수**: `floor(totalXp / 1600)`. `users.xp`에서 파생하며 완독 권수와 무관하다([nest.md §5.2.1](./nest.md), [backend.md §7.2](./backend.md)).
- **성 노출**: 1,600 XP 경계 통과 세리머니와 둥지 주기 정보에서 표시한다. 별도 성 선반이나 완독 책 기반 성 카운터는 두지 않는다.
- **읽은 책**: `user_books(status='completed')`를 표지·별점·소감·완독일과 함께 읽은 책 탭에 전시한다. 전체 공개 정책은 유지한다(§5.8.2).
- **정렬/필터 (#513·#795)**: 읽은 책 탭에서 최근순(기본)·별점순·제목순 정렬과 **별점 다중선택 필터**를 제공한다. ⭐ 필터 버튼(활성 개수 배지) → **바텀시트**: `★5·★4·★3·★2·★1·무평점` 다중선택(OR), **즉시 반영 + 초기화 + 완료**. 버킷 = 정수 floor(`4.5→★4`, `0.5~1.x→★1`), `무평점`은 **1급 옵션**(OCR 임포트 등 무평점 대량 유입 대비). 빈 선택 = 전체. (구 `⭐4점 이상` 단일 토글 대체)

#### 5.8.2 내 서재 (읽는 중 / 완독 / 중단 / 관심 책)

| 섹션 | 내용 |
|---|---|
| 읽는 중 | 진행 중인 책 목록 + 진척률. 탭 → 활성 책 전환 가능 |
| 완독 | 완독 책 목록. 별점·소감·완독일 표시. XP 주기 성과는 별개 |
| 중단 | **읽다 그만둔 책** (`user_books.status='aborted'`, #593). 표지+제목+저자+중단 시점 진척률. 카드 탭 → 책 상세. **"다시 읽기"**(`DataStore.myBooks.resume` → status='reading' 복귀, `current_page` 보존, **활성 책으로 승계**해 홈에 즉시·새로고침 후에도 표시 — #1203) 제공. 완독·XP 주기와 무관 |
| 관심 책 | 찜한 책 (`wish_books`). 표지+제목+저자+교보문고 링크+"지금 읽기 시작"(→ 책 등록). 기본 비공개 — `users.wishlist_public=true`로 설정 시 타인에게도 공개([feed.md §5.7.2](./feed.md)) |

- **내 서재 목록 3열 그리드화**: 현재 탭의 책 목록을 표지 중심 3열 그리드(표지 아래 제목 작게)로 표시하며, 탭 시 기존 `BookDetailModal`로 진입한다.
- **읽는 중 카드 라벨 (#1224)**: 표지 아래 상태 라벨은 **진행률 `현재쪽/전체쪽` 형태**(예 `23/320쪽`, 0쪽 포함) — 전체 쪽수 미상일 때만 "읽는 중". 구 "미완독" 라벨은 읽는 중 책에 오해 소지가 있어 제거. 완독(별점/완독)·중단(중단) 라벨은 기존 유지.
- **탭별 문장·감상 노출**: 책 목록 아래에 "💬 이 책들의 문장·감상" 섹션을 두고, 현재 탭에 속한 책들의 저장 문장만 최신순으로 표시한다.
- **다른 사람 책장 구경 가능** (전체 공개 정책). 타인 한 문장은 [feed.md §5.7.1](./feed.md) 페이지 블라인드 적용
- **타인 프로필 = 전체 페이지 (v7.2, #3·4·5)**: 피드 @닉 탭 → 풀스크린 프로필. ① 헤더(닉·완독/읽는중/스트릭/XP·팔로우 버튼) ② **책장 6권+더보기**(읽은/읽는중 필터 토글) ③ **❤️ 읽고 싶어하는 책**(대상 user의 `wishlist_public=true`일 때만 노출 — [feed.md §5.7.2](./feed.md)) ④ 공개 한 문장. **책 카드 탭 → 그 사람의 그 책 평점·후기·한 문장 드릴다운**(`users.bookContrib`). DataStore: `users.publicShelf`/`users.publicWishlist`/`bookContrib` ([backend.md §7.2](./backend.md))
- 관심 책 추가: 소셜 피드 한 문장 → 책 상세 → "관심 책에 추가" (`DataStore.wishBooks.add`)
- **책 중단 동선 (#593)**: 읽는 중 책의 상세(`BookDetailModal`)에서 **"읽기 중단"** → `DataStore.myBooks.abort(ubId)` → status='aborted'. 중단 책은 "읽는 중"·"완독"에서 빠지고 **"중단" 탭**으로 이동한다. 진척(`current_page`)은 보존되어 "다시 읽기"(`resume`) 시 그대로 이어진다. **`resume`은 status='reading' 갱신에 더해 그 책을 활성 책(`active_user_book_id`)으로 승계**(#1203) — 홈이 즉시 이 책을 띄우고 새로고침 후에도 '읽는 중'에 남는다(중단 탭에 잔류하지 않음). 양 어댑터 동일. 중단은 **삭제가 아니며**(책장에서 사라지지 않음) 완독 카운트·XP 주기에 영향 없다.
- **책 완전 삭제 동선 (#1195, Edgar 피드백)**: 잘못 담은 책을 되돌릴 수 없게 **영구 삭제**. 책 상세(`BookDetailModal`)에 중단과 **구분되는 3차 danger 텍스트 버튼 "이 책 삭제"** → `DataStore.myBooks.remove(ubId)`. 모든 상태(읽는 중·중단·완독)에서 노출된다. **확인창은 기록 유무를 알린다**: 한 문장이 N개 있으면 "한 문장 N개도 함께 삭제…", 문장 없이 읽은 진척만 있으면 "읽은 기록도 함께 삭제…", 아무 기록도 없으면 단순 확인. 삭제 시 그 책의 문장·세션도 함께 사라진다(Supabase는 `sentences`·`reading_sessions` FK `on delete cascade`, localStorage는 `user_book`에 내포). **활성 책을 삭제하면 남은 reading 책으로 active 승계**([nest.md §5.3](./nest.md)) — 홈이 빈 상태로 떨어지지 않는다. 중단(되돌리기 가능)과 삭제(영구)는 별개 액션이다.
- **책 완독 시 active 승계 (#1217)**: 활성 책을 완독(`DataStore.myBooks.complete` → status='completed')하면 그 책이 활성으로 남지 않게 **남은 '읽는 중' 책으로 active 승계**(없으면 null) — abort(#643)·resume(#1203)·remove(#1195)와 같은 불변식(`active_user_book_id` 는 항상 reading 책 또는 null). 누락 시 완독책이 활성으로 남아 홈이 완독책을 '읽는 중'으로 계속 띄우고 완독 탭에도 동시 노출되던 버그. 양 어댑터 동일. **DB 강제(#1217 재발 후, `45_active_book_invariant.sql`)**: 클라 픽스는 구버전 탭·APK를 못 덮어 재발 → 불변식을 트리거로 강제(status 이탈 시 자동 승계 + active 를 non-reading 으로 쓰는 시도 교정). JS 승계는 게스트(localStorage)·즉시 UI 용으로 유지.

#### 5.8.3 완독 별점 + 소감 (v7 신설)

완독 세리머니([nest.md §5.4](./nest.md)) 직후 입력:
- **별점** 0.5~5, **0.5 단위(반별점)** (선택, `user_books.rating`) — 별 탭은 정수, 별 좌측 절반 탭은 0.5
- **별점 접근성**: 선택 전 5개 별도 패널 배경과 **3:1 이상 비텍스트 대비**로 식별되어야 한다. 각 별 터치 영역은 최소 **44×44px 수준**이며, 키보드 포커스 윤곽과 선택값(`aria-checked`)을 제공한다.
- **완독 소감** 텍스트 (선택, `user_books.review_text`)
- 완독 목록·다른 사람 책장에 표시

- 한 문장 타임라인 (날짜 desc, 페이지/문장 + `my_note` 있으면 함께). 스포일러 블라인드 적용
- **한 문장 사후 감상**: 각 한 문장에 긴 감상(`my_note`)을 **작성 시점 이후 언제든 추가·편집**. 한 문장은 짧게(인용 ≤200자), 감상은 넉넉하게(권장 500자). 타임라인 항목 탭 → 감상 입력 (`DataStore.sentences.setNote`). 본인만 작성, 본인 프로필·책 상세에서만 표시. UX 참고: [북모리 캡쳐 분석](../bookmory-capture-analysis.md) (인용 위 감상 입력·페이지 메타·사후편집)
- **교보문고 링크**: `https://search.kyobobook.co.kr/search?keyword={isbn}` — "교보문고에서 보기 →" (Phase 1+ 어필리에이트 파라미터)
- **책 소개(description) 표시 (#530, #489 후속)**: 책 상세에 `📚 책 소개` 섹션. **DB `books.description` 우선**(있으면 그대로), 없을 때만 `fetchBookDesc()`(알라딘 프록시 ISBN 단건) **실시간 폴백**(쿼터 절약). 둘 다 없으면 섹션 생략. DB 값은 `myBooks` → `book.description` 으로 전달.
- **함께 읽으면 좋은 책 (#496)**: 책 상세 하단에 관련 도서 가로 캐러셀(표지·제목·저자). 표지 탭 → 찜(`DataStore.wishBooks.add`). 데이터: `DataStore.books.related(book)` → `data.js recommendRelated`.
  - **Phase 0 소스**: worker `POST /api/related`(LLM, `{isbn,title,author}` → `{books:[{isbn,title,author}]}`)가 ISBN-13 포함 후보를 제시 → 클라 `filterRelatedCandidates` 가 **books DB의 ISBN과 정확 일치(+정규화 제목 일치)** 일 때만 통과시키는 **ISBN 환각 필터**로 지어낸 책을 버리고 실제 책만 노출. LLM ISBN 은 신뢰하지 않으며 제목 prefix 매칭은 쓰지 않음. 매칭 0이면 섹션 자체를 숨김(빈 캐러셀·허위 없음).
  - **신뢰 카피 차등 (§5.8.6 정직성)**: Phase 0은 실제 '함께 읽은 사람들' 집계가 없으므로 **"N명이 함께 읽었어요" 류 허위 카피 금지** — "함께 읽으면 좋은 책"으로만 표기. 실제 공동독서 집계('이 책 읽은 사람들이 읽은 책')는 **Phase 1**에 Supabase `user_books` 기반 RPC로 강화(backend.md §7.9). 관련: [backend.md #489 환각 필터](./backend.md)
- **Export (v7.4, #315 — 북모리 양식 벤치마킹)**: Markdown 다운로드. **책 메타**(제목·저자/역자·출판사·쪽수·ISBN·표지) + **완독 정보**(별점·소감·완독일, 완독 시) + **한 문장**(`### p.N · YYYY-MM-DD` + `> 문장` + 감상). 날짜는 `created_at` 복원(미파싱 시 생략). 단일 책·책별 일괄 Markdown은 **UTF-8 BOM**을 포함해 Windows 자동 인코딩 감지에서도 한글이 깨지지 않아야 한다. **책 소개(description)는 후속**(알라딘 ItemLookUp OptResult 확장, #302 합류). Phase 2 이미지 카드
- 책 삭제(#1195): 상세 하단 3차 "이 책 삭제" 버튼 → 기록 유무 확인창 → **hard delete**(문장·세션 cascade, 활성 책이면 active 승계). 상세 동선은 §5.8.2 "책 완전 삭제 동선"

#### 5.8.8 한 문장 모아보기 (v7.1, #171)
홈([nest.md §5.1](./nest.md)) "오늘의 한 문장" 섹션의 "전체 N개 보기" → 모달. **전체 / 책별 / 좋아요** 필터 + **읽었음 누적 카운터**. 문장 탭 → 책 정보(BookInfoModal).

- **내 한 문장 → 책 상세로 이동 (v7.5, #439, 프로필에서 제거)**: 프로필 본문의 "내 한 문장" 섹션(최신 10개 + "더 보기")은 제거. 내 한 문장은 **프로필 → 내서재 → 읽고 있는 책 클릭 → 책 상세**(BookDetailModal)에서 그 책의 문장 + 참새 대화 기록과 함께 확인한다(companion.md §3). 좋아요 필터는 이 §5.8.8 모달에서 계속 확인 가능.
- **좋아요(❤️) 스탯 탭 → 좋아요한 문장 목록 (#510→#641)**: 프로필 stats row 의 `❤️ 좋아요 N` 을 탭하면 이 모달이 **'좋아요' 필터로 열린다**(`RG_openCollection({filter:'fav'})`). 좋아요 필터는 내 문장뿐 아니라 **내가 좋아요한 타인 문장까지 포함** — `claps.list()` 의 sentence 임베드를 내 문장 목록에 머지(타인 문장은 책 제목·페이지·문장 표시). 빈 상태: "아직 좋아요한 문장이 없어요 · 소셜 탭에서 마음에 드는 문장에 ❤️를 눌러보세요". (#641: ~~책갈피~~ 폐기, stats 라벨·식별자 모두 `좋아요`/`claps`로 단일화)
- **내 한 문장 검색 (v9 신설, #1007)**: 모아보기 모달 상단(필터 탭 아래)에 **검색 입력창**. 내 문장이 쌓이면(읽었음 수십~백 개) 스크롤로는 특정 문장을 못 찾으니 키워드로 즉시 좁힌다 — 모은 문장이 검색 가능한 **개인 독서 위키**가 되는 첫 단계(김문 님 피드백).
  - **범위 = 내 문장만** (`listMine()` 결과). 타인 공개 문장·전체 검색은 비대상(별도 기능 + 저작권 #1008).
  - **매칭 = 정규화 substring**(소문자·공백 정리한 부분일치). 대상 필드: 문장 `text` + 감상 `note`(my_note) + 책 제목 `bookTitle` + 저자 `author`. 퍼지(Fuse) 대신 substring — 한글에서 예측 가능하고 오매칭이 없다. 규모(유저당 ≤약 100문장, [backend.md sentences])라 **클라 필터로 즉시·오프라인**, 백엔드/검색 인덱스 불필요.
  - **기존 필터와 합성(AND)**: 검색어는 전체/책별/좋아요 탭 위에 겹쳐 적용한다(예: '책별' + 검색 → 책별 그룹 안에서 매칭만, 매칭 0인 책 그룹은 숨김). 빈 검색어 = 현행 동작 그대로.
  - **상태·문구**: 입력 즉시 반영(클라 동기 — 디바운스 불요). 매칭 0 → "검색 결과가 없어요". placeholder = "내 문장·감상·책 검색".
- **내 문장에게 묻기 — 독서 위키 Q&A (v10 신설, #1007)**: 같은 모달 상단에 **찾기 ↔ 🤖 내 문장에게 묻기** 모드 토글. '묻기'는 키워드 검색이 못 잡는 질문 — "내가 외로움에 대해 모은 문장?", "이 문장들 관통하는 주제?", **"다른 책에서 비슷한 생각 한 부분?"**(책 가로지르기, 여인채 님 아이디어 #919) — 에 LLM이 답한다. 검색이 *위치*라면 묻기는 *해석·연결*. 김문 님 피드백("LLM에게 질문도")의 실현.
  - **범위 = 내 문장만** (`listMine()` 결과, 검색과 동일 풀). 타인·전체 문장은 비대상. 문장 0개면 '묻기' 토글 미노출(비활성).
  - **그라운딩(환각 가드 핵심)**: 답은 **내가 모은 문장들에만 근거**한다. 워커 system 프롬프트가 (1) 근거 문장·책 제목을 함께 밝히고 (2) 문장에서 못 찾으면 지어내지 말고 정확히 "모은 문장에서는 못 찾았어요"라고만 답하며 (3) 문장에 없는 사실·다른 책 내용을 새로 만들지 않도록 강제한다. RAG/임베딩/벡터DB 불필요 — 규모(유저당 ≤약 100문장 ≈ 2-3K 토큰)라 **내 문장 전체를 한 프롬프트에 통째로** 넣어 **질문당 1콜**(Gemini Flash 텍스트, 무료 티어). 호출 경로·계약은 [backend.md §7.9.2](./backend.md).
  - **프라이버시**: 내 문장→LLM 전송은 재키 companion(`/api/companion`)이 이미 하는 선례와 같은 서버리스 프록시 경로(키 서버 보관, 클라 노출 0). **내 문장만** 보내므로 타인 발췌 수집/저작권(#1008) 위험은 낮다.
  - **UI·상태**: 질문 입력(textarea, ⌘/Ctrl+Enter 전송) + 보내기 → 로딩("문장을 살펴보는 중…") → 답 표시(근거 문장/책 포함). 첫 진입 시 **예시 질문 칩**(위 3종)으로 무엇을 물을 수 있는지 안내. 호출 실패 시 "답하기 실패 — 잠시 후 다시", 미설정(데모)이면 안내 문구. `wiki_ask` 이벤트 트래킹(문장 수·질문 길이).
  - **비대상(보류, #1007 후속)**: 자동 주제·소재 태깅/클러스터링, 전체(타인 공개) 검색. ('묻기'가 모은 문장에 한해 LLM 탐색을 이미 제공하므로 자동 태깅은 별개 후속.)
