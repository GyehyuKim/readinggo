### 5.8 프로필(내서재) 탭

```
┌─────────────────────────────────────┐
│ @계휴  · 통계 뱃지        [⚙️ 설정]   │
│ 🏰 ×3   🔥 21   ⚡ 1,240   완독 3권  │   ← 성 컬렉션 + 스탯
├─────────────────────────────────────┤
│ 🏰 성 컬렉션 (완독 3권)               │
│ [데미안] [82년생…] [불편한 편의점]    │   ← 완독 책 = 성 카드
├─────────────────────────────────────┤
│ 📖 현재 읽는 중  사피엔스 5/300p      │
│ [검색창]                             │
│ 📖 어린왕자 72/160p · 진행중          │
│ 📗 데미안 228/228p · 완독 ⭐4         │
│ 관심 책 (N)  [+ 책 추가]             │
└─────────────────────────────────────┘
```

- 상단: 닉네임·한 줄 소개(편집), 팔로워/팔로잉, **🏰 성 개수·🔥 스트릭·완독 권수** 스탯 + 검색 (한 줄로 압축, 높이 절약)
- **(v7.2 QA5, #205)**: 상단과 중복되는 중간 스탯 카드(둥지레벨/완독/스트릭/XP) **제거**, **XP 표기 전면 제거**(⚠️ 표시만 — XP 개념은 systems.md/승원 합의), **성 컬렉션을 프로필 최상단**으로, 성 카드 별점 **유효숫자**(4→4.0)
- "현재 읽는 중" 카드 탭 → [nest.md §5.3](./nest.md) 활성 책 전환 시트
- 책 카드 탭 → 책 상세
- ⚙️ 설정 → 닉네임 변경(고유·중복검사), **한 줄 소개(bio) 편집 (v7.2, #226)** — `users.bio`(≤100자), 헤더 소개 문구에 반영(미설정 시 placeholder), **내 데이터 내보내기(JSON, #172)**, **운영자 문의**(폼 → `inquiries` 저장 → admin 대시보드, [backend.md §7.2](./backend.md)), 알림 시간(Phase 2+), 로그아웃

#### 5.8.1 성(🏰) 컬렉션 — [nest.md §5.2.1](./nest.md) 전시

- 완독 1권 = 🏰 1개. `user_books(status='completed')` **파생** (별도 테이블 없음, [backend.md §7.3](./backend.md))
- **완독은 두 형태로 남는다**: ① 🏰 **성 카운트**(성취 척도, 둥지 탭) ② **표지 이미지 책장 전시**(영구 컬렉션). 완독이 성으로만 치환되지 않는다
- **표지 책장 = 소셜 대표 화면**: 타인이 내 프로필을 열면 보는 핵심 화면 — 완독 책 표지가 컬렉션처럼 누적 전시 (전체 공개, §5.8.2)
- **성 선반**: 완독 책 카드 그리드. 각 카드 = 표지 + 별점 + 완독일
- 둥지 탭 상단 `🏰 ×N` 미니 배지가 이 선반으로 연결

##### 5.8.1.1 책장 상세 (v7.4, #312)

- 프로필의 성 컬렉션 선반은 **최근 10권만** 미리보기.
- "🏰 성 컬렉션 (완독 N권)" **헤더 우측에 `책장 상세 →` 버튼**.
- 진입 시 **전체 완독 책을 5열 N행 그리드**로 북커버와 함께 표시.
- 그리드 상단 컨트롤:
  - **검색**: 제목·저자
  - **정렬**: 최근순(완독일) · 별점순 · 제목순
  - **필터**: 별점 구간(예: ⭐4+), 기간 등
- 카드 탭 → 책 상세(별점·소감·한 문장). 데이터: `castles`(완독 user_books 파생, 클라 측 정렬/필터).

#### 5.8.2 내 서재 (읽는 중 / 완독 / 관심 책)

| 섹션 | 내용 |
|---|---|
| 읽는 중 | 진행 중인 책 목록 + 진척률. 탭 → 활성 책 전환 가능 |
| 완독 | 완독 책 = 성 컬렉션과 동일 집합. 별점·소감 표시 |
| 관심 책 | 찜한 책 (`wish_books`). 표지+제목+저자+교보문고 링크+"지금 읽기 시작"(→ 책 등록) |

- **다른 사람 책장 구경 가능** (전체 공개 정책). 타인 한 문장은 [social.md §5.7.1](./social.md) 페이지 블라인드 적용
- **타인 프로필 = 전체 페이지 (v7.2, #3·4·5)**: 피드 @닉 탭 → 풀스크린 프로필. ① 헤더(닉·완독/읽는중/스트릭/XP·팔로우 버튼) ② **책장 6권+더보기**(읽은/읽는중 필터 토글; '보고싶은'은 wish RLS 비공개라 타인 비표시) ③ 공개 한 문장. **책 카드 탭 → 그 사람의 그 책 평점·후기·한 문장 드릴다운**(`users.bookContrib`). DataStore: `users.publicShelf`/`bookContrib` ([backend.md §7.2](./backend.md))
- 관심 책 추가: 소셜 피드 한 문장 → 책 상세 → "관심 책에 추가" (`DataStore.wishBooks.add`)

#### 5.8.3 완독 별점 + 소감 (v7 신설)

완독 세리머니([nest.md §5.4](./nest.md)) 직후 입력:
- **별점** 0.5~5, **0.5 단위(반별점)** (선택, `user_books.rating`) — 별 탭은 정수, 별 좌측 절반 탭은 0.5
- **완독 소감** 텍스트 (선택, `user_books.review_text`)
- 성 카드·완독 목록·다른 사람 책장에 표시

#### 5.8.4 책 상세 / Export

- 표지·제목·저자·진척 바
- **쪽수 미상 graceful (v7.2, #204)**: `total_pages`가 없으면(알라딘 itemPage 누락) 진척률 0·"쪽수 미상" 표기(1쪽=100% 버그 방지). 등록 시 BookInfoModal에서 **총 쪽수 수동 입력**(선택) 가능. *외부 교보 API는 무료 공식 소스 부재로 미채택 — 수동 폴백으로 대체*
- 한 문장 타임라인 (날짜 desc, 페이지/문장 + `my_note` 있으면 함께). 스포일러 블라인드 적용
- **한 문장 사후 감상**: 각 한 문장에 긴 감상(`my_note`)을 **작성 시점 이후 언제든 추가·편집**. 한 문장은 짧게(인용 ≤200자), 감상은 넉넉하게(권장 500자). 타임라인 항목 탭 → 감상 입력 (`DataStore.sentences.setNote`). 본인만 작성, 본인 프로필·책 상세에서만 표시. UX 참고: [북모리 캡쳐 분석](../bookmory-capture-analysis.md) (인용 위 감상 입력·페이지 메타·사후편집)
- **교보문고 링크**: `https://search.kyobobook.co.kr/search?keyword={isbn}` — "교보문고에서 보기 →" (Phase 1+ 어필리에이트 파라미터)
- **Export (v7.4, #315 — 북모리 양식 벤치마킹)**: Markdown 다운로드. **책 메타**(제목·저자/역자·출판사·쪽수·ISBN·표지) + **완독 정보**(별점·소감·완독일, 완독 시) + **한 문장**(`### p.N · YYYY-MM-DD` + `> 문장` + 감상). 날짜는 `created_at` 복원(미파싱 시 생략). **책 소개(description)는 후속**(알라딘 ItemLookUp OptResult 확장, #302 합류). Phase 2 이미지 카드
- 책 삭제: 길게 누름 → 확인 모달. 누적 기록 soft delete

#### 5.8.6 완독 후 AI 기능 (Phase 1+ 실호출, Phase 0 시뮬)

완독 후 책 상세에 AI 카드 노출. 호출 경로·비용·키 보호: [backend.md §7.9](./backend.md) (Gemini Flash 무료 + 서버리스 프록시).

① **AI 도서 추천 — 다음 책** *(개발: 계휴)*
  - 입력: 완독 책(제목·저자) + 내 한 문장 샘플(최대 5)
  - 출력: 다음에 읽을 책 3권 + 한 줄 이유. **나↔책 fit** 기반 (친구 매칭 아님 — 향후 ML 이전 고려)
  - CTA: 각 추천 책 "교보문고에서 보기 →"
  - Phase 0: 카테고리별 하드코딩 3권

② **AI 추출 책 — 나만의 추출 책**
  - 입력: 그 책에서 내가 남긴 한 문장 전체
  - 출력: 반응한 주제 TOP 3 + AI가 뽑은 "가장 인상 깊었던 한 문장" + 흐름 요약
  - Export: 이미지 카드(Phase 2) or Markdown
  - Phase 0: 한 문장 나열 + 고정 카피 시뮬

#### 5.8.7 무작위 한 문장 회상 (v7 신설)

- 프로필/내서재 상단에 **과거 내 한 문장 1개를 무작위로 다시 노출** ("그때 이런 문장을 남겼어요")
- 출처: 내 `sentences` 전체에서 무작위 1개 (`DataStore.sentences.random`). 책 제목·페이지·날짜 함께
- 탭 → 해당 책 상세. 기억·감성 강화 (북모리 "무작위 노트" 벤치마크, [ROADMAP](../ROADMAP.md) v7 채택 A)
- Phase 0: 로컬 `sentences` 무작위 / Phase 1: 서버 랜덤

#### 5.8.8 한 문장 모아보기 (v7.1, #171)
둥지 "전체 N개 보기" → 모달. **전체 / 책별 / 좋아요** 필터 + **읽었음 누적 카운터**. 문장 탭 → 책 정보(BookInfoModal).

- **내 한 문장 프로필 기본 노출 (v7.2, #228)**: 프로필에 내 전체 한 문장을 **최신 10개**까지 노출 + 소제목 우측 **"더 보기 →"**로 이 모달 진입. **좋아요한 문장은 별도 섹션이 아니라 이 모달의 "좋아요" 필터**로 확인(프로필 본문에서 좋아요 전용 섹션 제거).

#### 5.8.9 운영 대시보드 — is_admin 전용 (#161 / #190 A+B)
- 프로필 헤더 우측 📊 버튼 — `users.is_admin = true` 인 계정에게만 노출
- 클릭 → AdminDashboardModal: 가입자·**실사용자(NPC 제외)**·한 문장·완독·오늘 체크인 집계 + **최근 7일 추세**(아래) + 문의 목록(상태 토글·답장)

- **C단계 (v7.2, #190)**: **인기책 TOP**(등록·완독 수)·**활성 사용자 7/30일**(리텐션 프록시). 전역 집계라 RPC `admin_popular_books`/`admin_active_users`(SECURITY DEFINER + 본문 `is_admin()` 가드, `12_admin_insights.sql`). 리텐션 **코호트**·퍼널은 후속
- 보안: stats 집계는 anon count 허용 범위, 전역 RPC는 is_admin() 가드로 비admin 차단. is_admin 체크는 클라 UI 조건부 렌더

### 5.9 닉네임 규칙 (v7.1 — Model A: 닉네임 1개 통합) — [§4](./onboarding.md) E-1 참조

표시이름/아이디 구분 제거. 고유·변경가능 **닉네임 1개**(=`users.handle`)를 피드·프로필에 표시. 내부 식별은 불변 `users.id`(UUID) — 닉 변경해도 기록 유지. 저장 시 `display_name`도 동기화.

클라(`RG_VALIDATE`)/서버(DB CHECK) 동일 규칙:

```
정규식: ^[A-Za-z0-9_가-힣]{2,20}$   (v7.1: 대문자·20자 허용. 구 ^[a-z0-9가-힣_]{2,16}$ 대체)
금칙어: LDNOOBW + docs/readinggo/data/banned_ko.txt
중복: users.handle UNIQUE (설정에서 isHandleAvailable 사전 검사)
변경: 무제한
```
