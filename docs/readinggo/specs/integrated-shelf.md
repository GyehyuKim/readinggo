# 통합 서가 — 빈 서가 박멸 (#772 · #774)

> **신설 (2026-06-17)**: 오픈베타 헤비독서러 피드백에서 반복된 두 줄기를 하나로 묶은 spec.
> ① **능동** — 구매내역/서재 스샷 → 비전 OCR로 *내가 읽은 책* 복원 (#772, 태규).
> ② **수동** — 빈 책에 *남이 읽은 문장* 마중물 시드 (#774, 강민구).
> 공통 목표: **빈 서가/외로움 박멸 + lock-in**(흩어진 독서를 한 서가에 모아 자산화).
> **편집 정책**: spec-only. 코드는 후속 PR 시리즈(MVP→확장). Stack Lock 준수(§5).

## 1. 목적·전략

- **문제**: 현재 최대 약점은 "초빈이라 심심하다"(빈 서가). 신규 유저가 들어와도 채울 게 없어 다음 행동으로 안 이어진다.
- **lock-in 가설**(태규): "책을 한 군데서 안 읽는다(알라딘·밀리·도서관…). 기록이 흩어져 안 보인다. 한 곳에 모아주면 정 붙고 그게 lock-in."
- **두 줄기**:
  - **① 능동 복원(#772)**: 유저가 스샷 한 장 올리면 → 비전 OCR → 내 서가 좌르륵. "스샷 한 장 = 독서 인생 복원".
  - **② 수동 마중물(#774)**: 데이터/감상이 빈 책 → 웹/AI에서 "남이 읽은 한 문장"을 시드로 깔아 와글한 느낌. 공감 → 나도 쓰고 싶어지는 루프의 출발점.
- **#772 = 내가 읽은 책(능동) / #774 = 남이 읽은 문장(수동)** — "빈 서가 박멸"의 양쪽 절반.

## 2. 영혼 보존 제약 (불가침 — 태규 #18)

통합/대량 등록으로 가는 순간 "또 하나의 독서 기록 앱"(북적북적류)이 되기 쉽다. 리딩고의 영혼은 **"한 문장 + AI 대화 + 참새"**라는 가볍고 따뜻한 레이어. **통합은 거들 뿐, 그 영혼을 끝까지 지킨다.**

- 대량 등록이 메인 UX가 되지 않게: 복원은 *온보딩/빈 상태의 1회성 마중물*로 위치. 평상시 핵심 동선은 여전히 한 문장·짹·참새.
- 복원된 책도 "한 문장 남기기"로 자연스럽게 유도(영혼 레이어 진입점).

## 3. 비목표

- **외부 서점/이북 API 자동 연동 금지**: 개인 구매이력 비공개·약관 리스크. → 스샷 업로드로 우회(연동 협의·약관 이슈 0).
- **대량 데이터가 제품의 중심이 되는 것**(§2).
- **타인 블로그 원문 무단 스크랩·전재**(§4.2 — 저작권 가드).

## 4. ① 스샷 OCR 서가 복원 (#772)

### 4.1 동선 (UX)

1. **진입**: ⓐ 내서재 빈 상태 CTA "📸 스샷으로 서가 복원" ⓑ 온보딩 선택 단계(옵션).
2. **업로드**: 알라딘/밀리/교보 등 주문·서재 캡쳐 1장(다중은 후속). 기존 `OcrCropOverlay` 자르기 재사용 가능(선택).
3. **처리중**: 비전 OCR → 책 목록 추출(스피너 + "책 찾는 중…").
4. **검수 리스트**: 인식된 책 카드 목록(체크박스 + 제목·저자 인라인 편집). 매칭 실패분은 "미확인" 표기.
5. **일괄 등록**: 선택분을 서가에 등록. 완료 토스트 + "한 문장 남겨보기" 유도(영혼 진입점).

### 4.2 서버 — `POST /api/shelf-import` (OCR 스택 확장)

기존 `/api/ocr`(Upstage Document OCR + solar-pro3 보정)는 **자유 텍스트** 반환용이라, 서가 복원엔 **구조화 추출**이 필요 → 별도 엔드포인트(또는 `/api/ocr?mode=shelf`).

- 입력: `formData{ document: 이미지 }`. 재사용: Upstage `model=ocr` → raw 텍스트.
- 추가 단계: `callLLM`(solar-pro3, temperature 0.1)로 raw → **책 목록 JSON** 추출:
  `{ books: [{ title, author, status? }], partial: bool }`
  - 시스템 프롬프트: "구매내역/서재 캡쳐에서 책 제목·저자만. UI 텍스트(가격·배송·버튼)·중복 제거. 불확실하면 제외."
- 출력: `{ books, raw, partial }`. 키는 **서버 보관(클라 노출 금지)**, 8MB 제한(기존 `OCR_MAX_BYTES`).
- 실패/미설정: `503 { demo:true }` (기존 패턴) — 데모는 수동 추가 폴백.

### 4.3 매칭·미존재 처리

- 각 추출 책 → `DataStore.books.search(title)` + 저자 퍼지 매칭(isbn13 우선, 없으면 제목+저자).
- **매칭 성공** → 카탈로그 `Book`(표지·메타 확보).
- **매칭 실패** → "미확인 책"으로 보존(제목·저자 텍스트만). 검수 리스트에서 알라딘 검색 보조 버튼 제공(`/aladin`). 끝까지 미매칭이면 표지 없는 최소 책으로 등록 허용(메타는 후속 보강).

### 4.4 일괄 등록 (DataStore 계약 확장)

- 검수 후 선택분을 **배치 등록**. 신설 제안: `myBooks.addBatch(items[]) → UserBook[]` (없으면 `myBooks.add` 반복 — Phase 0 폴백).
- **기본 status**: 과거 독서 복원 맥락이라 후보는 `completed`(완독) 또는 `wish`. → **§7 미해결**(유저 선택 토글 권장: "다 읽었어요 / 읽고 싶어요").
- 게스트(미로그인)는 §7.7(backend.md) pending 흐름으로 로컬 보관 후 로그인 시 흡수.

### 4.5 폴백·실패 UX

- 인식 0건 → "스샷에서 책을 못 찾았어요. 더 선명한 캡쳐로 다시 시도하거나 직접 추가하세요." + 수동 추가 진입.
- 부분 인식(`partial`) → 인식분만 보여주고 "일부만 찾았어요" 안내.

### 4.6 프라이버시·분석

- 업로드 스샷은 **처리 후 미저장**(워커 메모리 경유, 영속 저장 없음). 원본은 분석·리플레이에 미포함.
- 이벤트: `shelf_import_started` / `shelf_import_extracted{count}` / `shelf_import_registered{count}` (analytics.md §3 필수=익명 집계).

## 5. ② 웹 리뷰 마중물 시드 (#774)

### 5.1 가치·원칙

- 핵심 가치는 *메타데이터 채우기가 아니라* → **외롭지 않게·와글한 느낌·마중물**. 남의 기록에 공감받고("당신도 그랬군요") 나도 쓰고 싶어지는 루프의 출발점.
- **진짜 유저 콘텐츠와 명확히 구분**(투명성).

### 5.2 ⚠️ 저작권·진정성 가드 (이 줄기의 핵심 리스크)

타인 블로그 원문을 긁어 그대로 노출하면 **저작권·인용범위 리스크**가 크다(#774가 직접 명시). 두 갈래:

- **A안 (권장) — AI 생성 시드**: 크롤링 원문 전재 대신, 공개 메타(책 소개·알라딘 description) + `callLLM`로 **"이 책을 읽은 가상의 한 문장·감상"을 생성**하고 **AI 생성임을 명시**(기존 #642 `'llm'` source → "AI" 칩 패턴 재사용). 저작권 리스크 0, 품질·안정성 통제 쉬움.
- **B안 (보류) — 실제 리뷰 인용**: 굳이 실제 블로그 인용 시 ⓐ 출처 URL 표기 ⓑ 짧은 스니펫(인용 범위 제한) ⓒ robots/약관 준수 ⓓ 법무 검토. → **법적 안전 전까지 보류.**

**권장**: P2는 **A안**으로 출발(리스크 0·즉시 가능). B안은 별도 결정·검토 후.

### 5.3 진정성 — "뻥 데이터"로 안 보이게

- **마중물 라벨**: `🌱 마중물`(또는 "함께 읽은 이웃") 배지로 *시드임을 투명 표기*. 가짜 유저를 *진짜 유저인 척* 위장하지 않는다(신뢰 붕괴 방지·계휴 "뻥으로 만들어놨던" 반성 반영).
- **자연 소멸**: 진짜 유저 문장이 쌓이면 시드는 **후순위→숨김**. 마중물은 *초기 공백만* 채운다.

### 5.4 서버 — `POST /api/seed` (A안)

- 트리거: 책 상세/서가 진입 시 그 책의 공개 한 문장(`sentences.byBook`)이 0건이면 시드 후보 요청.
- A안: `callLLM`로 책 제목·소개 기반 **시드 한 문장·감상 N개**(기본 3) 생성 — 다양한 톤, AI 명시. 캐시(책별, 재생성 억제).
- 출력: `{ seeds: [{ text, note?, source:'seed-ai' }] }`.

### 5.5 표시

- 책 상세/서가 "🌱 함께 읽은 이웃" 섹션 — 실제 유저 문장과 **시각 구분**(배지·톤 다운). 짹/대화 허용 범위는 §7 미해결(시드에 짹은 무의미 → 읽기 전용 권장, 대화는 가능).

### 5.6 데이터 모델

- `sentences.source`는 이미 존재(#642, `'llm'` 등). 시드용 `source='seed-ai'` + `is_seed=true` 구분, `user_id`는 시스템/`null`.
- RLS: 시드는 공개 read, 유저가 수정·삭제 불가(시스템 소유). 마이그레이션 별도 PR(.sql 수동 적용 필요).

### 5.7 리스크 매트릭스

| 리스크 | A안(AI 시드) | B안(실제 크롤) |
|---|---|---|
| 저작권 | 없음 | **높음**(출처·인용범위·약관) |
| 진정성("뻥") | 라벨+AI 명시로 통제 | 출처 있으면 진정성↑ |
| 품질·안정성 | LLM 통제·캐시 | 크롤 깨짐·잡음 유입 |

## 6. Stack Lock 정합 (CLAUDE.md)

- **OCR**: v8 결정(2026-06-11) — 웹 OCR 허용, Upstage Document OCR + solar-pro3, **키 서버 보관**. #772는 이 라인 **확장**(새 프레임워크 0).
- **크롤/생성**: 서버리스 워커(`worker/index.mjs`)에 엔드포인트 추가, 키 서버 보관. 클라 노출 금지.
- **DataStore 계약**: 피처 코드는 저장소 직접 호출 금지. `myBooks.addBatch`·시드 read는 계약 경유(backend.md §7.2 확장).
- **빌드**: React 18 CDN + Babel 유지. 신규 모듈은 #761 패턴(`shelf-import.js` 등 별도 파일 + `window.X`).

## 7. 미해결 질문 (코드 PR 전 사용자 확인)

1. **#774 방향**: A안(AI 생성 시드, 리스크 0) 우선 vs B안(실제 크롤, 법무 검토) — **권장 A**.
2. **#772 등록 기본 status**: 완독 vs 읽고 싶어요 — **권장: 유저 토글("다 읽었어요/읽고 싶어요")**.
3. **시드 문장 상호작용**: 짹/대화 허용 범위 — **권장: 읽기 전용(짹 무의미), 대화는 허용 검토**.
4. **다중 스샷**: MVP는 1장, 다중 업로드는 후속.
