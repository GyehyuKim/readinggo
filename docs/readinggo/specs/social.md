**이번 주 신규 독서 시작러 Top3**:

| 요소 | 규칙 |
|---|---|
| 집계 | 이번 주(월~일) 신규 `user_books` 생성 기준 상위 3권 |
| 표시 | 순위 + 책 제목 + "N명 시작" |
| Phase 0 | 하드코딩 시드 / Phase 1+ 주간 집계 쿼리 |

**한 문장 피드**:

| 섹션 | 규칙 |
|---|---|
| 범위 | **전체 사용자** — 친구 필터 없음. 모든 공개 한 문장이 시간순 노출 (`DataStore.sentences.feed`) |
| 카드 구성 | 아바타 + 닉 + 책 제목 탭(→ 책 상세) + **페이지(`p.N`)** + 한 문장 텍스트 + 리액션 |
| 리액션 | **짹 1종** (좋아요, ±1 토글). 내 한 문장에 받은 짹 = XP +1 ([systems.md §6.3](./systems.md)) |
| 책갈피 | 카드 우측 상단 🔖 → 관심 문장 저장 (`sentence_bookmarks`, [profile.md](./profile.md)) |
| 책 자세히 | 책 제목 탭 → [profile.md](./profile.md) 책 상세 |
| 본인 카드 | 짹·책갈피 비활성 |
| 빈 상태 | "아직 한 문장이 없어요. 오늘의 문장을 남겨보세요 🐦" |

#### 5.7.1 스포일러 — 페이지 기반 블라인드 (v7, SSOT)

수동 비공개(`is_private`) 폐기. 한 문장 노출은 **읽은 위치 기준 자동 블라인드**로 일원화. **이 절이 블라인드 규칙의 단일 출처** — 책 상세·프로필·마을은 이 규칙을 참조.

| 상황 | 처리 |
|---|---|
| 내가 **읽고 있는** 책의 한 문장 중, **내 현재 페이지보다 뒤** 페이지 | **블라인드** — "⚠️ 내가 아직 안 읽은 부분 · 탭하면 보기" |
| 내 현재 페이지 이하 | 그대로 노출 |
| 내가 **읽지 않는** 책 | **전체 공개** (블라인드 없음) |
| 내가 **완독**한 책 | **전체 공개** |

- **전역 토글**: 헤더 `🔓 스포일러 그냥 보기` ON/OFF. ON이면 블라인드 전부 해제(상관없이 보고 싶을 때).
- 적용 범위: **소셜 피드 · 책 상세 · 프로필 한 문장 모음 · 마을 한 문장** — 전 영역 동일 규칙.
- 판정 데이터: 뷰어의 `user_books.current_page`(해당 책) vs `sentences.page` 비교 (`DataStore.spoiler.myCurrentPage`). 저장 컬럼 없음.
