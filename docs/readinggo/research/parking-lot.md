## 1. 휴식코스(Pause) — 스트릭 동결 (#251, #126)

- **무엇**: 일정 기간 스트릭을 동결(휴식)해 다독자·바쁜 유저의 번아웃·이탈 방지.
- **왜 추가하려 했나**: 베타에서 "스트릭 끊김 = 이탈" 우려. 차감/위협 대신 보호 장치.
- **왜 뺐나**: 이번 코어는 **스트릭 자체가 목적이 아님** — "한 문장이 나를 끌어올리는 순간(고양감)"이 코어. 스트릭은 보조 지표일 뿐이라 휴식코스는 우선순위 낮음. 채찍 제거(완료)와 같은 맥락 — 스트릭 비중 자체를 낮춘다.
- **재개 조건**: 리텐션 데이터(PostHog)에서 스트릭 끊김이 실제 이탈 주원인으로 확인될 때. owner: 승원.

## 2. 멀티 디바이스 관리 + 실시간 동기화 (#191)

- **무엇**: 로그인 기기 목록 관리 + 여러 기기 실시간 상태 동기화.
- **왜 추가하려 했나**: 여러 기기 사용자 정합·보안.
- **왜 뺐나**: **지금은 이르다(Phase 2)**. 유저 규모·멀티기기 니즈가 검증되기 전. 현재는 단일 기기 흐름으로 충분.
- **재개 조건**: 활성 유저 증가 + 멀티기기 사용 패턴 확인 시. Phase 2.

## 3. 문의 LLM 자동 응답 (Hermes/전담 LLM) (#208)

- **무엇**: DB 문의 → LLM 자동 즉시 응답.
- **왜 추가하려 했나**: 운영 부하 감소, 즉시 응대.
- **왜 뺐나**: **메인 아님 + 당장 아님.** 코어(한 문장·파트너)와 무관한 운영 자동화. LLM 역량은 독서 파트너(#287/companion.md)에 집중.
- **재개 조건**: 문의량이 수동 처리 한계를 넘을 때. 코어 안정화 후.
