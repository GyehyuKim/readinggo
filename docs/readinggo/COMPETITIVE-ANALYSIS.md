> 작성: 2026-05-21
> 목적: 학기말 출시(Phase 2) 메시징·기능 우선순위·중기 로드맵의 입력 문서

> **추가 (2026-06-02)**: 실사용 55컷 정밀 분석 → [bookmory-capture-analysis.md](./bookmory-capture-analysis.md)

## 0. 요약 (Executive Summary)

1. **북모리(국내 250~500만 DL, 평점 4.8)는 의도적으로 소셜·게이미피케이션을 배제**한 1인 개발 4년차 제품. 시장 빈자리를 만든 장본인.
2. 글로벌에서 **Bookly = "Duolingo for Reading"** 으로 우리와 컨셉 정면 충돌. 단 한국 미진출 + 소셜 없음.
3. **Fable(3M 사용자)** = 100K+ 북클럽으로 폭발 성장한 소셜 독서 플랫폼. 한국 미진출.
4. **리더스(국내)** = 인스타그램형 팔로우 + 자체 북클럽 → 한국에서 가장 가까운 소셜 경쟁자. 게이미피케이션은 약함.
5. **시장 빈자리**: *고-게이미피케이션 × 고-소셜 × 한국어 × 1페이지 원칙* 조합은 비어있음 → ReadingGo의 좌표.
6. **친근감 3-tier 모델** (지인·독서모임원·플랫폼 유저)에서 우리는 **T1·T3 강 / T2 비어있음**. T2를 어떻게 채우느냐가 다음 결정의 핵심.
7. **결정**: 옵션 C (T2 mini — 같은 책 자동 클럽 패널만). **의도된 임시그룹(특정 책·일시·장소 모임)은 중기 과제로 보관**.

### 1.2 북모리 정밀 분석

**기본**
- 개발자: Tony Soft (1인 개발, 4년차 운영)
- 다운로드: 250만~500만 (Google Play 기준)
- iOS 글로벌 평점: **4.78 / 5** (73,000 평가)
- iOS 한국 평점: **4.8 / 5** (2,600 평가)
- 언어: 6개국어 글로벌 지원
- 플랫폼: iOS + Android (PC 없음)
- Instagram 팔로워: 24K
- MAU: 비공개

**수익 모델**
- Freemium: 무료 + 프리미엄 구독 (월 ₩3,900 / 연 ₩35,000)
- 프리미엄 혜택: **광고 제거** + 자동 동기화 + 통계 확장 + OCR 무제한 + 테마/폰트
- 일회 결제 옵션 부재 (사용자 불만 다수)

**기능 (그들이 잘하는 것)**
- 트래킹: 페이지·시간·진척, 내장 타이머 (시그니처), 독서 달력, 월/연간 통계, 일일·연간 목표
- 노트: Rich Text, 인용구, **OCR 텍스트 추출**, 무작위 노트
- 라이브러리: 컬렉션/시리즈, 맞춤 태그, 평점/리뷰, 바코드 스캔
- 개인화: 다크/라이트 + 커스텀 색상, PIN/생체인식 잠금
- 연간: Annual Rewind (매년 12월)

**의도적으로 안 하는 것** (개발자 브런치 명시)
> *"본질에 집중 — 소셜 기능을 의도적으로 배제"*
- 소셜·친구·피드 완전 부재
- 게이미피케이션 매우 약함 (캐릭터 칭찬·누적 시각화 정도)
- 클럽·그룹 독서 없음
- 챌린지·미션 없음

**부정 리뷰 — 우리 기회 5가지**

| # | 불만 | 빈도 | ReadingGo 대응 |
|---|---|---|---|
| 1 | **광고가 독서 흐름을 깬다** | 최다, 이탈 사례 다수 | **광고 없음** 정책 → 메타데이터 명시 |
| 2 | PC 버전 없음 | 자주 | 우리 웹앱(`docs/readinggo/`) 유지 — 본의 아니게 우위 |
| 3 | 노트 복사 불편 | 중 | Markdown Export 이미 구현 (IM-18) |
| 4 | 구독만 가능, 일회 결제 부재 | 중 | 학기 데모는 무료 → 향후 모델 차별화 |
| 5 | 평가 시스템 유연성 부족 | 약 | 우선순위 낮음 |

### 1.3 리더스 — 한국 내 가장 가까운 소셜 경쟁자

- 인스타그램형 팔로우 + 자체 북클럽 운영
- 게이미피케이션은 약함 → 여기가 우리 우위
- **우리와 다른 점**: 그들은 "팔로우 + 큐레이션 + 북클럽", 우리는 "스트릭 + 마을 + 한 문장"

### 1.4 북적북적 — 게이미피케이션 wedge 일부 침범

- "책 탑 쌓기" + 캐릭터 컬렉션 = 시각적 게이미피케이션
- 별점/메모만 — 통계·기록은 단순
- 소셜 없음
- **차별 포인트**: 그들은 "혼자 탑 쌓기", 우리는 "친구와 함께 짹"

## 2. 글로벌 시장 매핑

### 2.1 경쟁자 한 줄 정리

| 앱 | 사용자 규모 | 포지셔닝 | 소셜 | 게이미 | 한국 진출 |
|---|---|---|---|---|---|
| **Goodreads** | 1억+ (Amazon 소유) | 책 평점·리뷰 데이터베이스 | ✅ 깊음 (친구·그룹·DM) | 약 | △ (한국어 미흡) |
| **StoryGraph** | 수백만 | 데이터 분석 · 챌린지 | 중 (커뮤니티 챌린지) | 약 | ❌ |
| **Fable** | **3M, 폭발 성장** | **북클럽 + 챕터별 토론** · DM · 피드 | ✅ **매우 강함** | 중 (일일 스트릭·연간 목표 신설) | ❌ |
| **Bookly** | 수백만 | **"Duolingo for Reading"** 명시 | ❌ | ✅ **매우 강함** | ❌ |
| **Basmo** | 중규모 | "Reading Contract" + AI 챗 | 약 | 약 (배지 없음) | ❌ |
| **Litsy** | 중규모 | 책 모먼트 공유 + 챌린지 호스팅 | ✅ | 중 (챌린지) | ❌ |
| **Hardcover** | 신생 | StoryGraph 대안 | 중 | 약 | ❌ |

### 2.2 Bookly — 컨셉 정면 충돌 (가장 큰 위협)

> 그들의 슬로건: *"like Duolingo, but for reading"* — ReadingGo와 정확히 동일

**기능 깊이**
- 일일·월간·연간 목표 (시간 + 페이지)
- 연속일 스트릭
- 마일스톤 배지
- 마스코트 캐릭터
- 앰비언트 사운드 (집중용)
- 폴리시 인포그래픽
- 정밀 시간 추정 ("이 책 마치는 데 X시간 걸림")

**약점**
- **소셜 부재** — 모든 게이미가 1인용
- **타이머 시작·종료의 마찰** — "1페이지만 읽기"에 부적합
- **한국어 미지원 · 한국 책 데이터 없음**

**ReadingGo vs Bookly 비교**

| 영역 | Bookly | ReadingGo |
|---|---|---|
| 핵심 메카닉 | 스트릭 + 배지 + 마스코트 + 일일 목표 | 스트릭 + XP + 방패 + 둥지 + 리그 |
| 진입장벽 | **타이머 시작 → 측정 → 종료** (마찰 큼) | **"1페이지 OK"** (마찰 최소) |
| 콘텐츠 자산 | 인포그래픽 (개인) | **한 문장 수집 → 사회적 자산** |
| 소셜 | ❌ | ✅ 마을 + 피드 + 리그 |
| 한국어/한국 책 | ❌ | ✅ books.tsv |

**차별 포인트 명확**: ① 1페이지 원칙 ② 소셜 ③ 한국 시장

### 2.3 Fable — 소셜 독서의 표준이 된 신흥 강자

**기능 깊이**
- 100,000+ 북클럽 (장르별·테마별)
- 챕터별 비동기 토론
- 1:1 DM
- 클린 소셜 피드
- 일일 "I read today" 스트릭 (최근 추가)
- 연간 목표 (Fable Plus = $50/yr — 주간/월간 목표는 유료)
- Reader 배지
- 가벼운 팔로우 시스템

**ReadingGo vs Fable 비교**

| 영역 | Fable | ReadingGo |
|---|---|---|
| 그룹 형성 | **북클럽 가입 필요 (사전 결정)** | **자동 마을 그룹화 (수동적)** |
| 토론 | 챕터별 비동기 토론 | 한 문장 카드 + 리액션 칩 |
| 시간 부담 | 책 클럽 = 일정 동기화 부담 | 1페이지·1문장 = 부담 없음 |
| 한국어 | ❌ | ✅ |

→ **차별 포인트**: ① 클럽 가입 없이도 친구가 있음 ② 1페이지 진입장벽 ③ 한국 시장

### 2.4 다른 글로벌 앱

- **Goodreads**: 가장 큰 사회망. 추천 알고리즘 · 친구 · 그룹 · DM · 평점. 하지만 게이미피케이션은 약함. 한국어 미흡으로 국내는 약함.
- **StoryGraph**: 데이터 분석가 페르소나. 챌린지는 있지만 게이미피케이션 아님.
- **Basmo**: "Reading Contract" 약속 모델 + AI ChatBook Assistant (책 내용 요약·캐릭터 분석). 배지 없음.
- **Litsy**: 책 사진·인용 공유. 책별 dioama 생성. 사용자가 챌린지 호스팅 가능.

## 3. 시장 좌표 (Positioning Map)

```
                 Solo · Personal           Social · Communal
              ┌─────────────────────┬─────────────────────────┐
   Low        │  북모리              │  리더스, 그믐, 트레바리   │
   Game       │  북플립              │  Fable                  │
              │  Goodreads          │                          │
              │  StoryGraph, Basmo  │                          │
              ├─────────────────────┼─────────────────────────┤
   High       │  Bookly             │  ⚡ ReadingGo ⚡        │
   Game       │  북적북적            │  (한국 + 1페이지 원칙)    │
              │  Litsy              │                          │
              └─────────────────────┴─────────────────────────┘
```

**우리가 차지할 자리**:
- 한국어 시장에 **"고-게이미피케이션 + 고-소셜 + 1페이지 원칙"** 조합 부재
- Bookly = 고-게이미 / 노-소셜 / 영어권
- Fable = 노-게이미 (최근 일일 스트릭만) / 고-소셜 / 영어권
- 리더스 = 노-게이미 / 고-소셜 / 한국어
- ReadingGo = **고-게이미 + 고-소셜 + 한국 + 1페이지**

## 4. 친근감 3-tier 모델 (전략 프레임)

게이미피케이션이 통하려면 *"누구와 함께 하는가"* 가 명시되어야 함. 세 링별로 메카닉이 다름.

| Tier | 누구 | 결합 강도 | 적합 메카닉 | ReadingGo 현재 |
|---|---|---|---|---|
| **T1. 지인** | 친구·가족·동기 | 가장 강함 | 직접 가시성, 콕찌르기, 짝궁 스트릭 | ✅ 마을 둥지 + 모이 + 친구 상세 시트 |
| **T2. 독서모임원** | 같은 책 / 관심 그룹 | 중간 | 공동 목표, 챕터별 토론, 진도 동기화 | ❌ **사실상 비어있음** |
| **T3. 플랫폼 유저** | 모르는 독자 + NPC | 약함 | 리그·랭킹, 익명 응원, 피드 | ✅ 주간 리그 + 👏🥹🔖 + NPC |

### 4.1 현재 강약

```
T1 (지인):     ████████░░  강함
T2 (모임원):   █░░░░░░░░░  매우 약함  ← 최대 구멍
T3 (플랫폼):   ███████░░░  꽤 강함
```

### 4.2 친근감을 만드는 4가지 설계 레버

| 레버 | 효과 | T1 | T2 | T3 |
|---|---|---|---|---|
| **가시성** | 그냥 보임 → 인지 → 공감 | 마을 둥지 불빛 | 클럽 진도 막대 | 리그 순위 |
| **공동 목표** | 같이 짓는다 | 짝궁 스트릭 (?) | 도서관 짓기, 세계수 | 시즌 리그 |
| **가벼운 응답** | 부담 없는 한 번의 터치 | 모이 보내기 | 챕터 박수 | 👏🥹🔖 |
| **공유 콘텐츠** | 너의 한 문장이 나에게 | 친구 문장 카드 | 클럽 문장 모음 | 피드 + 이미지 카드 |

### 4.3 디자인 원칙 (v5 스펙에 박을 한 줄)

> **모든 게이미피케이션 기능은 "어느 tier의 친근감을 만드는가"를 명시한다. 어느 tier도 활성화하지 않는 기능은 의심하라.**

## 7. 결정 — T2 처리 방향

### 7.1 옵션 정의

| 옵션 | 작업량 | 효과 | 학기 일정 적합 |
|---|---|---|---|
| **A.** 풀 채움 (CL-01 클럽 + CL-02 공동자산) | 크다 | Fable·리더스 정면 대결 | ❌ (v6) |
| **B.** T1·T3 유지 (T2 비움) | 0 | 학기는 깔끔. Fable 한국 진출 시 늦음 | △ |
| **C.** T2 mini (자동 클럽 패널만) | 작다 (1~2일) | 효과 중간, 시간 효율 좋음 | ✅ |

### 7.2 채택: **옵션 C + 의도된 임시그룹은 중기 보관**

**Phase 2 (학기 출시) — 옵션 C 적용**
- `books.tsv` 기반으로 *"같은 책 읽는 사람들"* 자동 패널 생성
- 책 상세 화면 하단에 "이 책 같이 읽는 친구 N명" 표시 → 탭하면 그들의 한 문장 카드 리스트
- 클럽 가입 절차 없음 (Fable과 차별점 — 가입 없이도 노출)
- 공동 자산은 v6로 미룸

**중기 (v6 / 출시 후 1~3개월) — 의도된 임시그룹 추가**
- **특정 책 모임**: "이 책을 5월 21일~6월 4일에 함께 읽을 사람 모집" 형태
- **특정 일시**: 정해진 기간 동안만 활성, 끝나면 자동 아카이브
- **특정 장소** (선택): 오프라인 모임 연동 (트레바리·문토와 차별 영역)
- 트리거: 사용자가 만들거나, 책별 자동 큐레이션 ("이번 주의 책")

이 그룹은 T2의 *"가벼운 공동 목표 + 시간 제한"* 자석을 활용. Fable이 영구 북클럽이라면, 우리는 *팝업 북클럽*. 트레바리·문토는 오프라인이라 다른 결.
