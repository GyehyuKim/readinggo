# Pitch

교수님·투자자·팀원에게 30초 안에 설명하기 위한 세 가지 질문.

---

## 30초 요약

> **ReadingGo는 "하루 한 페이지, 한 문장"을 약속으로 잡는 독서 습관 앱이다.**
> 언어 학습 앱이 "하루 5분" 약속과 끊기지 않는 스트릭으로 외국어를 일상에 끼워 넣은 것처럼, 우리는 같은 엔진을 *책*에 이식한다.
> 타겟은 안 읽는 사람이 아니라, **읽고 싶은데 이어나가지 못하는 사람**이다.

---

## What's New?

> 기존에 없던 무엇을 새로 만드는가?

### 한 줄 답

**검증된 지속성 엔진(스트릭·방패·세리머니·둥지 진화)에 "오늘의 한 문장(모이) 강제 입력"을 결합해, 독서 행위 자체를 *읽음의 증명*으로 묶어내는 첫 번째 앱.**

### 세 가지 결합

1. **언어 학습 앱이 증명한 지속성 엔진을 독서로 이식.**
   - 스트릭 (연속일) / 방패 (회복 자산) / 세리머니 (confetti·보상 카드 3그리드) / 둥지 진화 5단계 (🪵 → 🪹 → 🏠 → 🏡 → 🏰) / 21시·23시 2단 알림.
   - 언어 학습·운동 앱이 외국어와 운동에서 검증한 메커닉이지만, *독서 카테고리에 진지하게 이식된 사례는 없다*. 기존 독서 앱은 책장 정리 도구이거나 구독형 콘텐츠 플랫폼이지, 매일 1회 행동을 만들어내는 엔진이 아니다.

2. **"모이" — 오늘 옮겨 적은 한 문장을 *강제로* 받는 구조.**
   - 페이지 입력만으로는 읽음을 검증할 수 없다. 한 문장(200자 이내)을 직접 적는 행위가 (a) 읽음의 자기 증명이며 (b) 사용자에게 누적되는 자산이다.
   - 누적 문장은 책별로 Markdown export 가능. 떠나도 가져갈 수 있다는 신뢰가 lock-in의 윤리적 부담을 상쇄한다.

3. **NPC + 단방향 팔로우로 첫날부터 비어있지 않은 소셜.**
   - 가입 즉시 2명의 NPC(책읽는곰돌이·활자라쿤) 자동 팔로우. 마을 둥지 그리드에 불빛이 켜져 있다.
   - 친구 요청·수락 마찰 없음. 단방향 팔로우 + 🪱 모이(콕찌르기)는 친구당 일 1회 제한 → 압박감 차단.
   - 소셜 피드는 *모든 사용자의* 공개 모이가 흐르는 발견 채널. 좋은 문장은 발견되고, 좋은 책은 옮겨 다닌다.

### 기존 제품과의 포지셔닝

| 제품 | 본질 | 우리와의 차이 |
|---|---|---|
| **밀리의 서재 / 리디북스** | 구독형 전자책 플랫폼 | 콘텐츠 *접근*은 풀지만 *지속*은 안 푼다. 우리는 콘텐츠를 안 팔고, 매일의 행동만 책임진다 |
| **북적북적 / 북아일랜드** | 독서 기록·완독 자랑 SNS | 다 읽은 사람의 *전시* 도구. 우리는 *읽고 있는 중*인 사람의 매일에 개입한다 |
| **Goodreads / StoryGraph** | 평점·리뷰·추천 데이터베이스 | 읽은 책을 정리하는 도구. 우리는 *읽기 행동의 매일 1회 트리거*가 본질 |
| **언어 학습 · 운동 앱** | 다른 카테고리의 지속성 엔진 | 동일 엔진 원리를 *책*에 진지하게 이식한 사례가 시장에 비어 있다 |

**우리가 점유하는 빈 곳: "매일 1회 책 행동을 만들어내는, 콘텐츠를 안 파는 앱."**

## So What?

> 그래서 누구에게 왜 중요한가?

세 계층에서 답한다.

### ① 개인 — *읽고 싶었지만 이어지지 않던 사람의 일상이 바뀐다*

타겟은 명확하다. **책을 사놓고 3페이지에서 멈춘 적이 있는 사람.** 이 사람은 의지가 없는 게 아니라, 하루의 호흡에 1페이지가 끼어들 자리가 없을 뿐이다.

ReadingGo가 약속하는 변화:

| 가입 직후 | 1주 후 | 1개월 후 | 6개월 후 |
|---|---|---|---|
| 첫 책 등록 + 첫 문장 기록 + 세리머니 (4분) | 스트릭 7일 → 첫 방패 획득. "끊고 싶지 않다"는 감각이 생긴다 | 1권 완독 + 둥지 진화 🏰. 누적 문장 30~50개 — 책 한 권의 엑기스가 손에 남는다 | Export Markdown 1권 + 2권 → 자기만의 인용집. 읽는 *사람*이 된다 |

핵심은 *"오늘 읽었다"는 사실*이 매일 만들어진다는 것. 부담 없이, 강요 없이, 1페이지·1문장으로.

### ② 시장 — *비어있는 카테고리, 검증된 메커닉*

- **국내 독서 인구는 줄고 있지만, "독서를 늘리고 싶다"는 욕구는 줄지 않는다.** 문체부 국민독서실태조사에 따르면 성인 독서량은 매년 감소하나, *"독서를 더 하고 싶다"* 응답률은 60%대로 유지. 욕구는 있는데 행동이 안 따른다 — 우리가 푸는 정확한 문제다.
- **습관 형성 카테고리는 자기계발 앱 시장에서 검증된 매출 구간이다.** 언어 학습·명상(Calm·Headspace)·운동(Strava) 등 인접 카테고리가 조 단위 시총을 형성했고, 동일 엔진을 새 카테고리에 이식하는 전략은 반복적으로 작동한다.
- **콘텐츠를 안 사기 때문에 유닛 이코노믹스가 가볍다.** 우리는 책을 라이선스하지 않는다. 사용자가 가진 책을 읽도록 도울 뿐. 콘텐츠 비용 0 → 잔존만 증명되면 구독형 모델 진입 가능 (Phase 2+).

### ③ 사회 — *AI 시대에 인간이 읽는다는 행위의 가치*

LLM이 책을 대신 요약하고, 짧은 영상이 긴 호흡의 콘텐츠를 잡아먹는 시대에 — *직접 읽고, 한 문장을 옮겨 적는다*는 행위는 그 자체로 가치가 있다.

우리는 책을 요약해주지 않는다. 추천하지도 강요하지도 않는다. 사용자가 *자기 손으로 옮겨 적은 한 문장*만이 매일 누적된다. 이 누적은 AI 시대의 인간이 잃고 있는 능력 — *긴 호흡으로 읽고 자기 언어로 옮기는 능력* — 의 작은 복구 행위다.

BIZ.69911 강의 주제(AI 기반 비즈니스 진화)와의 연결점도 여기 있다. AI를 *대체*로 쓰는 제품이 넘치는 시점에, 우리는 *AI가 못 대신해주는 행위*를 매일 1회 만들어내는 제품을 만든다.

## Why You?

> 왜 이 팀이 이걸 해야 하는가?

### 한 줄 답

**우리 자신이 그 타겟이기 때문이다.**

세 명의 팀원(계휴·승원·윤지)은 모두 *"읽고 싶었는데 못 이어간"* 사람이다. 책장에 못 끝낸 책이 있고, 작년에 사놓은 책이 새것 상태로 있고, 좋은 문장을 보고 "옮겨 적어야지" 하다가 잊은 경험이 있다.

자기 자신의 문제를 푸는 팀은 다른 팀이 못 보는 것을 본다.

### 우리가 본 것 (다른 독서 앱이 못 본 것)

| 우리가 본 진실 | 시장에 박혀 있는 가정 |
|---|---|
| **부담은 동기를 죽인다.** 1페이지여도 충분히 의미 있다 | "하루 1시간 독서"가 디폴트. 작은 단위를 인정 안 함 |
| **읽음의 증명은 페이지가 아니라 *옮겨 적은 한 문장*이다** | 페이지 입력·완독 체크만으로 기록 |
| **혼자 읽는 사람도 *함께 읽는 감각*이 필요하다** | 친구 요청·수락 마찰 + 빈 친구 목록 |
| **떠날 자유가 머무를 이유를 만든다** | 데이터 export 없음. 락인 일변도 |
| **첫날의 마찰이 평생 잔존을 결정한다** | 사전 설문·튜토리얼·권한 요청 3종 세트 |

이 다섯 가지 통찰은 사용자 인터뷰가 아니라 *우리 자신의 실패 경험*에서 나왔다. ReadingGo의 모든 의사결정 — 사전 설문 제거, 모이 강제, 단방향 팔로우, NPC 동행, Export 우선 — 은 이 통찰의 직접적 산물이다.

### 실행 신뢰의 근거

- **Phase 0 데모가 이미 작동한다.** 스펙 v4.4 + `docs/readinggo/index.html` Phase 0 빌드 — 책 등록·모이 입력·세리머니·둥지 진화·마을·소셜·내서재·활성 책 전환·날짜 시뮬레이터까지 클릭으로 흐른다. 542권 books.tsv, 4명 NPC 시드, Moneygraphy 두 폰트 적용 완료.
- **자기충족적 스펙.** 외부 참조 없이 시스템 재구현이 가능한 1100줄 단일 스펙 문서를 유지하고 있다. 팀이 흩어지거나 LLM이 바뀌어도 동일하게 빌드된다.
- **Phase 분리 명확.** Phase 0(localStorage 데모) → Phase 1(Supabase + pg_cron + RLS) → Phase 2(Android APK + FCM)로 단계가 잘려 있고, 각 단계의 백엔드 스키마(users·books·user_books·reading_sessions·sentences·streak·shield_log·follows·claps·pokes·wish_books·sentence_bookmarks)가 명세 완료.

타겟이 우리 자신이고, 데모가 이미 손에 있고, 다음 단계의 설계도가 이미 그려져 있다.
*우리가 안 만들면 누가 만드는가* — 라고는 말하지 않겠다. 그저: **우리가 만드는 게 가장 빠르다.**
