## 0. 한 줄 전략

**"읽고 싶은데 이어나가지 못하는 사람"에게, 스토어에서 첫 세리머니까지 3분 안에 도달시킨다.**

지표로는: 설치 → 첫 문장 세리머니 도달율. 이게 aha moment이고, 여기 도달한 사용자만 D1에 돌아온다.
GTM의 모든 항목(리스팅·온보딩·카피)은 이 퍼널의 어느 단계를 개선하는지로 평가한다.

## 1. 퍼널 정의

```
스토어 노출 → 리스팅 조회 → 설치 → 게스트 진입 → 책 등록 → 첫 문장 → 세리머니(aha) → 로그인(저장) → D1 복귀
```

| 단계 | 무기 | 현재 상태 |
|---|---|---|
| 노출→조회 | 앱 이름·아이콘 | 아이콘 리디자인 완료(2026-06-28, 세이지 참새+골드 스파크) ✅ |
| 조회→설치 | 스크린샷·설명 | **미제작** — §3 |
| 설치→책 등록 | 게스트 우선 + 빈 상태 카피 | 뼈대 좋음, 카피 부정문 — §4 P1 |
| 책 등록→첫 문장 | 책+문장 한 화면(C-2) | 완성 ✅ |
| 첫 문장→세리머니 | confetti+보상 3카드 | 완성 ✅ |
| 세리머니→로그인 | CTA | Google 고정 — §4 P1 |
| 로그인→D1 | 스트릭·알림 | 알림 Phase 2 — 초기엔 스트릭 심리만 |

## 2. 포지셔닝·메시지 위계 (모든 표면 공통)

pitch.md 30초 요약을 소비자 언어로 접은 것. 모든 카피는 이 3층에서 꺼내 쓴다.

1. **약속(헤드라인)**: 하루 한 쪽, 한 문장.
2. **방법(서브)**: 책을 더 쉽게, 더 깊게 읽도록. — 공유 태그라인(#1092)과 동일
3. **증거(바디)**: 스트릭·둥지·내 문장 컬렉션·같은 책 타인 한 문장·사진 한 장으로 문장 추출

금지: "독서 혁명", "당신의 삶을 바꾸는" 류 과장. 기능이 아니라 결과를 거창하게 말하는 순간 신뢰를 잃는다.

## 3. Play 스토어 리스팅

### 3.1 텍스트

- **앱 이름 (30자)**: `리딩고 — 하루 한 문장 독서 습관`
- **짧은 설명 (80자)**: `책을 더 쉽게, 더 깊게. 하루 한 쪽, 한 문장으로 독서 습관을 만들어요.`
- **전체 설명 골격** (4000자, 확정 카피는 별도 커밋):
  1. 문제 공감 — 사놓고 3페이지에서 멈춘 책 (MANIFESTO의 타겟 정의 그대로)
  2. 방법 — 오늘 읽은 자리에서 한 문장만 옮겨 적기. 그게 오늘의 성공
  3. 쌓이는 것 — 스트릭·둥지 진화·내 문장 컬렉션(Markdown 내보내기 = 떠나도 가져감)
  4. 같이 — 같은 책을 읽는 타인의 한 문장
  5. 도구 — 책 사진 한 장으로 문장 추출(OCR), 바코드로 책 등록

### 3.2 그래픽 에셋

| 에셋 | 규격 | 상태 |
|---|---|---|
| 앱 아이콘 | 512×512 PNG | 원본 1024×1024 있음(`assets/icon.png`) → 리사이즈만 |
| 피처 그래픽 | 1024×500 | **미제작** — 참새+태그라인, DESIGN.md 팔레트(세이지·종이) |
| 폰 스크린샷 | 최소 2장, 권장 5장+ | **미제작** — §3.3 |

### 3.3 스크린샷 5장 (처음 3장만 스크롤 없이 보임 — 순서가 전부)

1. **홈(둥지+책+스트릭)** — "이 앱이 뭔지" 한 장. 둥지 5단계 시그니처 노출
2. **한 문장 남기기 → 세리머니** — 핵심 행동과 보상
3. **내 문장 컬렉션** — 쌓이는 자산 (lock-in의 윤리적 반전: 내보내기 가능)
4. **같은 책 타인 한 문장 / 피드** — 첫날부터 비어있지 않은 소셜
5. **OCR 사진→문장** — 마찰 제거 도구

각 장 상단에 캡션 한 줄(메시지 위계 §2에서). 데모 데이터는 실제 문학 문장 사용(저작권: 단문 인용 범위).

## 4. 온보딩 개선 (코드 실측 기반)

현행 여정(게스트 진입→빈 상태→책 등록→문장→세리머니)의 뼈대는 유지. **새 화면 추가 없음** — 카피와 연결만 수선.

| 우선 | 항목 | 위치 | 내용 |
|---|---|---|---|
| **P0** | "클로즈베타 · 지인 초대" 제거 | app.js 로그인 화면 하단 | 공개 출시와 모순. 심사 혼선 위험 |
| **P1** | 빈 상태 카피를 가치제안형으로 | nest.js 온보딩 블록 | "아직 읽는 책이 없어요"(부정문) → 약속 선언 + 등록 CTA. 스토어 유입은 앱을 모르고 옴 |
| **P1** | 세리머니 가입 CTA 확장 | ceremony | `Google로 계속` 고정 → 로그인 화면(구글/카카오/이메일) 경유. 한국 안드로이드=카카오 주력 |
| **P2** | 바코드 스캔 온보딩 노출 | 검색 화면 | 실물 책 유저에게 더 빠른 경로가 이미 구현돼 있는데 안 보임 |

## 6. 채널 (설치를 어디서 데려오나)

Phase 순서대로. 유료 광고는 리텐션 검증 전 금지(새는 바가지에 물 붓기).

1. **베타 → 공개 전환**: 기존 베타 사용자(지인·문의 등록자)에게 스토어 링크 공지. 첫 리뷰 시드 — 초기 별점이 노출 알고리즘을 좌우
2. **공유 루프(이미 구현)**: 철학 한 줄 + OG 미리보기(#1092) 초대 공유, 문장 카드 공유 — 제품 내장 채널이 주력
3. **독서 커뮤니티 시딩**: 트레바리·독서모임·북스타그램에 "한 문장" 결과물(문장 카드)로 진입 — 광고가 아니라 산출물 공유
4. **ASO 반복**: 리스팅 키워드(독서 습관·독서 기록·책 스트릭) 검색 순위 관찰 → 카피 조정

## 7. 지표 (측정 없으면 GTM 아님)

analytics.md 이벤트 위에서. 북극성: **주간 문장 수**(WSS — 진짜 읽고 기록한 양).

| 퍼널 지표 | 목표(초기 가설) |
|---|---|
| 설치→첫 문장 세리머니 | > 40% |
| 세리머니→로그인 전환 | > 50% |
| D1 복귀 | > 30% |
| D7 스트릭 생존 | > 15% |

숫자는 첫 코호트에서 보정. 지표가 무너지는 단계가 다음 스프린트의 유일한 주제.
