# MANIFESTO

> **하루 한 페이지, 한 문장에서 시작해요.**

---

## 우리가 존재하는 이유

세상에는 두 종류의 독서 앱이 있다.
하나는 "안 읽는 사람을 읽게 만들겠다"고 약속하는 앱이고, 다른 하나는 다 읽은 책을 기록·전시하는 앱이다.

전자는 동기가 없는 사람을 끌어오려다 실패하고, 후자는 이미 다 읽은 사람만 남는다.

우리는 **그 사이의 빈 곳**, 즉 *"읽고 싶은데 이어나가지 못하는 사람"* 을 위해 존재한다.
이 사람은 책을 사두고, 첫 챕터를 폈고, 다시 펴는 데 실패한 경험이 있다. 의지가 없어서가 아니라, **하루의 호흡에 1페이지가 끼어들 자리를 만들지 못해서** 그렇다.

ReadingGo는 그 자리를 만든다. "오늘 30분 독서"가 아니라, **오늘 1페이지·1문장**. 그것만 지키면 오늘은 성공이다.

우리가 답해야 하는 질문은 단순하다.
*어떻게 하면 사람이 자기 의지로는 못 만들어내는 매일의 행동을, 앱이 대신 만들어줄 수 있는가.*

## 우리가 풀려는 단 하나의 시나리오

> 책을 사놓고 3페이지에서 멈춘 사람이, **오늘 1페이지를 더 읽는다.**
> 그리고 내일도, 모레도 읽는다.

이 시나리오는 우리가 만드는 모든 화면·기능·문구의 평가 기준이다.

이 시나리오에 기여하지 않는 것은 만들지 않는다.
이 시나리오를 방해하는 것 — 부담, 죄책감, 비교, 마찰 — 은 제거한다.

핵심 루프는 세 단계로 압축된다.

```
앱 밖에서 1페이지 읽기 → 앱에서 페이지+한 문장 입력 → 스트릭+XP+마을 불빛
```

페이지 입력만으로는 읽음을 검증할 수 없다. 그래서 **"모이"** — 오늘 책에서 옮겨 적은 한 문장 — 를 강제한다. 이것이 (a) 읽음의 증명이고 (b) 사용자에게 남는 자산이다.

목표 페이지는 설정하지 않는다. 1페이지만 읽어도 오늘은 성공.
부담을 만드는 모든 장치를 의도적으로 제거하는 것이 우리의 설계 원칙이다.

## 우리가 만드는 것 / 만들지 않는 것

### 만드는 것

| 구분 | 무엇 | 왜 |
|---|---|---|
| **지속성 엔진** | 스트릭 / 방패 / 21시·23시 알림 / 미참여 자동 정산 | 의지가 끊기는 지점을 시스템이 받아준다 |
| **모이 (오늘의 문장)** | 200자 한 문장 강제 입력 + 페이지 추적 | 읽음의 증명이자 떠나도 가져가는 자산 |
| **The Path (둥지 탭)** | 세션 단위 노드 + 참새 캐릭터의 둥지 진화 5단계 | 진척의 가시화 — 보일 때 멈추기 어렵다 |
| **리딩 빌리지 (마을 탭)** | 친구+NPC 둥지 그리드, 불빛 ON/OFF, 🪱 모이 보내기 | 혼자가 아니라는 감각, 단방향 팔로우 + 일 1회 넛지 |
| **소셜 피드** | 모든 사람의 오늘의 모이가 흐르는 공개 피드 + 짹(좋아요)·🔖책갈피 | 좋은 문장은 발견되고, 좋은 책은 옮겨 다닌다 |
| **NPC 동행** | 가입 즉시 따라오는 책읽는곰돌이·활자라쿤 외 2명 | 마을이 비어 있지 않다. 첫날부터 함께 읽는다 |
| **Markdown Export** | 누적 문장을 책별로 다운로드 | 데이터를 가둬두지 않는다. 떠날 자유가 lock-in의 윤리적 부담을 상쇄 |

### 만들지 않는 것

| 안 만드는 것 | 왜 |
|---|---|
| **목표 페이지 설정** | "오늘 30페이지" 같은 숫자가 부담을 만들고 이탈을 부른다 |
| **사전 설문·튜토리얼** | 모든 마찰 제거. 첫 화면에서 책 등록까지 한 흐름 |
| **AI 요약·추천 슬롭** | 책을 대신 읽어주지 않는다. 읽는 사람의 손에 남는 한 문장이 본질 |
| **양방향 친구 요청** | 단방향 팔로우. 부담 없이 따라가고 끊을 수 있다 |
| **푸시 떡칠** | 알림은 하루 2회(21시·23시). 22시 이후 사용자 설정 차단 |
| **결제·구독 (Phase 0/1)** | 습관이 먼저 형성되어야 한다. 비즈니스 모델은 잔존이 증명된 뒤 |
| **안 읽는 사람 끌어오기** | 우리 타겟이 아니다. 의지가 0인 사람을 1로 만드는 앱은 망한다 |

## 우리의 Core Values

### 1. 1페이지 원칙 (One-Page Principle)

> 가장 작은 단위가 가장 큰 자산이다.

목표는 1페이지. 모이는 1문장. 알림은 1일 1회.
모든 수치는 "사용자가 가장 피곤한 날에도 할 수 있는가"를 기준으로 결정한다.

### 2. 호흡의 자리 (Make Room, Not Demands)

> 우리는 시간을 빼앗지 않는다. 끼어들 자리만 만든다.

앱은 책을 읽는 동안 꺼져 있다. 앱은 *읽고 난 뒤 30초*에만 켜진다.
"앱 안에서 책을 읽게 한다"는 모든 유혹을 거부한다.

### 3. 떠날 자유 (Right to Leave)

> 모은 것은 사용자의 것이다.

문장은 언제든 Markdown으로 다운로드된다.
가둬두지 않는 신뢰가 머무를 이유를 만든다.

### 4. 함께 읽는 감각 (Felt Companionship)

> 혼자 읽는 사람도, 혼자 느껴지지 않게.

NPC는 첫날부터 마을에 산다. 친구가 없어도 둥지에 불빛이 켜진다.
사회적 연결은 *압력이 아니라 온도*로 작동한다.

### 5. 게이미피케이션의 절제 (Discipline over Dazzle)

> 게이미피케이션은 빌리되, 게임이 되지 않는다.

스트릭·방패·XP·세리머니는 도구일 뿐. **책을 읽었다는 사실 자체를 흐릿하게 만드는 어떤 장치도 넣지 않는다.**
배지 남발, 광고형 보상, 도파민 루프는 만들지 않는다.

### 6. 자기충족적 스펙 (Self-Sufficient Specification)

> 문서만 보고 시스템이 돌아갈 수 있어야 한다.

스펙·매니페스토·피치는 외부 참조 없이 자기충족적으로 작성한다. 팀이 흩어져도, LLM이 바뀌어도, 다음 사람이 펴서 그대로 만들 수 있도록.

## 우리가 성공했다는 신호

### Phase 0 — 데모 (2026-05-16)

- [ ] 4분 안에 *책 등록 → 모이 입력 → 세리머니 → 마을 → 소셜 → 내서재 → 활성 책 전환* 전 흐름이 끊김 없이 흐른다
- [ ] 시연 후 평가자가 "내 친구한테 보여주고 싶다"고 말한다
- [ ] 슬로건이 시연 끝나고 30분 뒤에도 기억에 남는다

### Phase 1 — Supabase MVP

- [ ] **D7 잔존율 40%↑** — 가입 7일 뒤에도 1세션 이상 기록한 사용자 비율
- [ ] **평균 스트릭 5일↑** — 활성 사용자의 중앙값
- [ ] **모이 누적 1만 문장↑** — 사용자가 만든 자산의 총량
- [ ] 사용자가 자발적으로 "친구 찾기"로 다른 사람을 데려온다

### Phase 2 — Android APK + FCM (학기말)

- [ ] **D30 잔존율 20%↑** — 한 달 후에도 살아남는 비율 (독서 앱 업계 평균 5~10%)
- [ ] **완독 사용자 100명↑** — 1권을 끝까지 읽고 🏰 참새의 성에 도달한 사람
- [ ] Export 사용 빈도가 1인당 월 1회↑ — 사용자가 자기 자산을 *실제로 쓴다*

### 우리가 실패했다는 신호 (안 가야 하는 길)

- 스트릭을 위해 사용자가 *읽지 않은 채로* 페이지를 입력한다 — 모이의 검증 강도가 부족하다는 뜻
- 피드가 자기과시·인용 자랑 공간이 된다 — 본질이 흐려졌다는 뜻
- 마을이 압박감이 되어 "친구 보기 싫어서 앱을 안 켠다"는 피드백이 나온다 — 온도 조절 실패
